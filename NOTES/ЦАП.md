Любой канал ЦАПа может быть подключен к встроенным периферийным устройствам, таким как компаратор, операционный усилитель и АЦП. В этом случае выходной канал ЦАП может быть отключен от выходного пина DACx_OUTy, и соответствующий GPIO может быть использован для другой цели.
# Включение канала ЦАП
В любом ЦАПе есть всего 2 канала, и их можно включить через EN1 EN2 в регистре DAC_CR. Для включение нужного канала требуется время. Поэтому нам нужно ждать пока бит DACxRDY(x-номер канала) в DAC_SR не станет равен 1. Пока это не произойдёт мы не можем ничего преобразовывать!

# Преобразование в ЦАП

## Формат данных ЦАП
Данные которые мы хотим записывать зависят от настроенной нами конфигурации.
- 8 битные данные выровненные по правому краю записываются в `DAC_DHR8Rx[7:0]`(x-номер канала). Если используется двойной ЦАП, то вторые данные записываются в `DAC_DHR8Rx[15:8]`

- 12 битные данные выровненные по левому краю записываются в `DAC_DHR12Lx [15:4]`. Если используется двойной ЦАП, то вторые данные записываются в `DAC_DHR12Lx[31:20]`
![[Pasted image 20241124161510.png]]
- 12 битные данные выровненные по правому краю записываются в `DAC_DHR12Rx[11:0]`. Если используется двойной ЦАП, то вторые данные записываются в `DAC_DHR12Rx[27:16]`

## Знаковые/беззнаковые данные
Входные данные в ЦАП по умолчанию беззнаковые: 0x000 соответствует минимальному значению, а 0xFFF — максимальному значению для 12-битного режима.

ЦАП также может обрабатывать знаковые входные данные в формате дополнительного кода. Это делается путем установки бита SINFORMATx в регистре DAC_MCR.

Когда бит SINFORMATx установлен, старший значащий бит (MSB) данных, записанных в регистры DHRx, инвертируется при копировании в регистр DAC_DORx, и интерфейс ЦАП может принимать знаковые данные (формат Q1.15, Q1.11 или Q1.7). Регистр DAC_DHR12Lx может использоваться для хранения 16-битных знаковых данных в регистрах удержания данных. 12 старших битов 16-битных данных используются для выходных данных ЦАП, и старший бит инвертируется. Четыре младших бита просто игнорируются.

|SINFORMATx|DATA written to DHRx|DATA transferred to DORx|
|---|---|---|
|0|0x000|0x000|
|0|0xFFF|0xFFF|
|1|0x7FF|0xFFF|
|1|0x000|0x800|
|1|0xFFF|0x7FF|
|1|0x800|0x00|

## Как происходит преобразование
Мы не можем записывать данные в DAC_DORx напрямую, этот регистр только для чтения поэтому нам нужно пользоваться DAC_DHRx.

Когда регистр `DAC_DORx` загружается содержимым регистра `DAC_DHRx`, аналоговое выходное напряжение становится доступным через время $t_{SETTLING}$, которое зависит от напряжения питания и аналоговой нагрузки на выходе.

$R_L$ и $C_L$ - это сопротивление и ёмкость нагрузки соответственно

$\pm0.5$ LSB это изменению выходного напряжения ЦАП на половину наименьшего значащего бита. Лучше это показать на примере
Предположим, у вас есть 12-битный ЦАП с опорным напряжением 3.3 В. В этом случае:

- Полный диапазон напряжения: 0 В до 3.3 В.
- Разрешение ЦАП: 3.3 В / 4096 (2^12) = 0.8056 мВ на LSB.

±0.5 LSB означает изменение выходного напряжения ЦАП на половину значения LSB. В данном примере:

- 0.5 LSB = 0.8056 мВ / 2 = 0.4028 мВ.

Таким образом, ±0.5 LSB означает изменение выходного напряжения ЦАП на ±0.4028 мВ.

Короче говоря считай что нам всегда нужно примерно 3 микросекунд.

## Выходное напряжение ЦАП
Выходное напряжение цап вычисляется по формуле
$$DAC_{output}=V_{Ref}\cdot\frac{DOR}{4096}$$

# Инициализация преобразований
Если установлен управляющий бит TENx, преобразование может быть инициировано внешним событием(таймер-счетчик, внешняя линия прерывания). Управляющие биты `TSELx[3:0]` в регистре DAC_CR определяют, какое из 16 возможных событий инициирует преобразование.
Внешние тригеры можно узнать в *таблице 183 Reference manual stm32g4* 

Каждый раз, когда интерфейс DAC обнаруживает фронт нарастания на выбранном источнике триггера, последние данные, сохраненные в регистре DAC_DHRx, передаются в регистр DAC_DORx. Регистр DAC_DORx обновляется через три тактовых цикла dac_hclk после возникновения триггера. Когда выбран программный триггер, передача из регистра DAC_DHRx в регистр DAC_DORx занимает только один тактовый цикл dac_hclk.

Если выбран программный триггер, преобразование начинается после установки бита SWTRIG. Бит SWTRIG сбрасывается аппаратно после загрузки регистра DAC_DORx данными из регистра DAC_DHRx.
> [!attention] 
> SWTRIG нужен только в случае TENx=1 и TSELx=00, если же TEN=0, то ЦАП будет автоматически начинать преобразовывать напряжение как только на DHRx будут записаны данные 

> [!important] 
> Бит TSELx[3:0] не может быть изменен, когда установлен бит ENx 

## Запросы DMA
Каждый канал ЦАП имеет возможность работы с DMA. Для обслуживания запросов DMA каналов ЦАП используются два канала DMA.

Когда происходит внешний триггер (но не программный триггер) при установленном бите DMAENx, значение регистра DAC_DHRx передается в регистр DAC_DORx после завершения передачи, и генерируется запрос DMA.

В двойном режиме, если оба бита DMAENx установлены, генерируются два запроса DMA. Если требуется только один запрос DMA, должен быть установлен только соответствующий бит DMAENx. Таким образом, приложение может управлять обоими каналами ЦАП в двойном режиме, используя один запрос DMA и один канал DMA.

Поскольку передача данных из DAC_DHRx в DAC_DORx происходит до запроса DMA, первые данные должны быть записаны в DAC_DHRx до возникновения первого триггерного события.
### Недогрузка(underrun) DMA
Запрос DMA ЦАП не ставится в очередь, поэтому если второй внешний триггер приходит до подтверждения первого внешнего триггера (первого запроса), новый запрос не генерируется, и устанавливается флаг underrun DMA канала x DMAUDRx в регистре DAC_SR, сигнализируя об ошибке. Канал ЦАП x продолжает преобразовывать старые данные.

Программное обеспечение должно сбросить флаг DMAUDRx, записав 1, сбросить бит DMAEN используемого потока DMA и повторно инициализировать как DMA, так и канал ЦАП x для правильного возобновления передачи. Программное обеспечение должно изменить частоту триггера преобразования ЦАП или уменьшить нагрузку на DMA, чтобы избежать повторного underrun DMA. Наконец, преобразование ЦАП может быть возобновлено путем включения как передачи данных DMA, так и триггера преобразования.

Для каждого канала ЦАП x также генерируется прерывание, если его соответствующий бит DMAUDRIEx в регистре DAC_CR включен.
### Режим двойных данных DMA
Когда контроллер DMA используется в нормальном режиме, передаются только 12-битные (или 8-битные) данные одним запросом DMA. Поскольку ширина AHB составляет 32 бита, два 12-битных данных могут передаваться одновременно. Для использования этого режима установите бит DMADOUBLEx регистра DAC_MCR.

Запрос DMA ЦАП генерируется каждые два внешних триггера (кроме программных триггеров) при установленном бите DMAENx:

1. Когда обнаруживается первый триггер, значения регистров DAC_DHRx и DAC_DHRBx передаются в регистры DAC_DORx и DAC_DORBx. Фактические данные ЦАП загружаются в регистр DAC_DORx. Затем генерируется запрос DMA. DMA записывает новые данные в регистры данных DAC_DHRx и DAC_DHRBx.
2. Когда обнаруживается следующий триггер, фактические данные ЦАП загружаются в регистр DAC_DHRBx. Этот второй триггер не генерирует никаких запросов DMA. Бит DORSTATx указывает, какие данные DOR фактически загружены во вход ЦАП.

Функция underrun DMA также поддерживается в режиме двойных данных DMA.

В режиме двойных данных DMA запросы могут обрабатывать только один канал ЦАП. Для использования двух выходных каналов в режиме двойных данных DMA каждый канал DMA должен быть настроен отдельно.

Для изменения режима работы ЦАП с режима двойных данных (Double Data Mode) на режим одиночных данных (Single Data Mode) или наоборот, необходимо выполнить следующие условия:
- Нужный канал ЦАПа должен быть выключен(ENx=0)
- Бит DMAENx должен быть сброшен

# Пилообразный сигнал в ЦАП
Выбор триггера сброса и выбор триггера инкремента для генерации пилообразного сигнала осуществляется через управляющие биты STRSTTRIGSELx и STINCTRIGSELx соответственно. Сопоставление STRSTTRIGSELx аналогично TSELx. См. раздел 22.4.2: Пины DAC и внутренние сигналы для сопоставлений TSELx, STRSTTRIGSELx и STINCTRIGSELx.
