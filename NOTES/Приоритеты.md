## Base
Еще одним очень важным моментом, связанным с прерываниями, является наличие приоритетов. Каждому прерыванию (кроме некоторых системных) может быть присвоен приоритет от 0 до 15.  При этом приоритет работает в обратную сторону - чем ниже число, тем больше приоритет. Изначально все прерывания имеют приоритет 0, то есть максимально возможный.

Если программа остановлена прерыванием с низким приоритетом, и в момент выполнения обработчика прерывания было вызвано прерывание более высокого приоритета, то точно также, как и с главной программой, выполнение приоритета с более низким приоритетом прервется, выполнится обработчик прерывания более высокого приоритета, после чего продолжится выполнение обработчика с более низким приоритетом.

Если в момент выполнения прерывания с более высоким приоритетом появится прерывание с более низким, это прерывание будет ожидать завершения другого, с более высоким приоритетом, для того, чтобы выполнится.

Если станет вопрос об одновременном выполнении двух прерываний с одинаковым приоритетом, то предпочтение будет отдано тому, которое имеет меньший номер в NVIC.

Для установки приоритета прерывания используется функция NVIC_SetPriority, которая принимает на вход прерывание, для которого изменяется приоритет, и значение нового приоритета.
```c
NVIC_SetPriority(EXTI2_IRQn, 2);
```
