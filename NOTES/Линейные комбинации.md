# Linear combinations

Линейная комбинация - это способ собрать один объект из других, просто масштабируя их и складывая.

Так давайте возьмём два вектора $arrow(u), arrow(v)$
![[Pasted image 20251223220006.png|400]]
Их сумма будет представлять простейшую линейную комбинацию, т.к. мы получили новый вектор путём сложение двух других:
$$arrow(z) = arrow(u) + arrow(v)$$

И если мы возьмём два скаляра $c_(1) = frac(1, 2), c_(2) = frac(1, 3)$ то мы получим другую линейную комбинацию полученную из тех же векторов
![[Pasted image 20251223220201.png|400]]
$$arrow(z) = c_(1) arrow(u) + c_(2) arrow(v) = frac(1, 2) arrow(u) + frac(1, 3) arrow(v)$$

И мы не ограничиваемся масштабированием векторов в пределах $[0;1]$ мы можем брать любые вещественные скаляры и получать новые вектора - новые линейные комбинации. Тем самым мы можем получить любой вектор $arrow(z)$ в виде линейной комбинации векторов $arrow(u)$ и $arrow(v)$ взятые со скалярами:

$$arrow(z) = c_(1) arrow(u) + c_(2) arrow(v)$$

Одним из важных линейных комбинаций в картезианской системе являются единичные вектора $arrow(i) = (0, 1)$ и $arrow(j) = (1, 0)$, вместе с этим может использоваться $arrow(k) = (0,0,1)$ если у нас 3х мерное пространство
Благодаря этим единичным векторам можем записать любой position vector в картезианской системе в виде линейной комбинации этих векторов
![[Pasted image 20251223220840.png|500]]
Для обозначения единичных векторов ещё используется обозначение
$$arrow(e_(1)), arrow(e_(2)), arrow(e_(3)), dots, arrow(e_(n))$$

Единичные вектора также являются базисными векторами
> [!help] 
> **Базисные векторы** — это набор **независимых векторов**, с помощью которых можно **записать любой вектор пространства как линейную комбинацию** этих векторов.
> **Базис** - это набор **независимых элементов** которые можно комбинировать вместе чтобы выразить другой элемент.
> Примером не векторного базиса может быть RGB где базисами являются Red Green Blue и мы выражаем другой цвет в количественном выражении базисов Red Green Blue от 0 до 255. При этом мы можем использовать все те же свойства векторов для RGB так как в них выполняется все свойства базисов
> $$(3 3,6 6,9 9)_(R G B) = 3 3 R + 6 6 G + 9 9 B$$ 

> [!note] 
> Линейные комбинации могут использоваться не только с векторами, но также с любыми математическими объектами которые могут быть суммированы и масштабированы, например на картинке ниже представлена линейная комбинация синусов и косинусов
> ![[Pasted image 20251223221425.png|700]]

# Линейные операции

Линейные операции это такие математические операции, что сохраняют линейные комбинации.
То есть если применить линейную операцию к линейной комбинации объектов, то результатом будет **та же комбинация от образов операции объектов**.

То есть линейные операции должны иметь все свойства нужные для того чтобы они сохраняли линейные комбинации.

Так если у нас есть некоторая операция $T: RR^(n) -> RR^(m)$, то такая операция является линейной только в случае если выполняется три свойства линейности:
$$l

T (arrow(x)+arrow(y)) = T (arrow(x)) + T (arrow(y)) "(аддитивность)"  \

T (alpha arrow(x)) = alpha T (x) "(однородность)"  \
 
T (alpha arrow(x) + beta arrow(y)) = alpha T (arrow(x)) + beta T (arrow(y)) "(общая формула)"$$


Так примером линейной операции является производная

$$l

(f+g)' = f' + g'  \

(alpha f)' = alpha f'  \

(alpha f + beta g)' = alpha f' + beta g'$$

То же самое относится и к операции интеграла или [преобразованию Лапласа](Преобразование%20Лапласа.md)

Но например синус или косинус не является линейной функцией
$$l
sin (x+y) != sin (x) + cos (y)  \
  
                                 \
 
sin (x+y) = sin (x) cos (y) + sin (y) cos (x)$$
