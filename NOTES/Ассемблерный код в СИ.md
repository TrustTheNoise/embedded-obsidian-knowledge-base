# Ассемблерные вставки

Некоторые вещи в Си просто не реализованы. Так например в Си отсутствуют средства для прямого доступа к **специальным регистрам процессора** (например, регистрам управления или статуса в ARM Cortex-M) и некоторым **низкоуровневым инструкциям**, таким как `CLZ` (Count Leading Zeros) или операции с битовыми полями, которые критичны для оптимизации или работы с аппаратными особенностями микроконтроллеров. Кроме того, Си не предоставляет механизмов для **прямого управления памятью** на уровне аппаратных адресов или специфичных для архитектуры операций, которые могут потребоваться при работе с периферией или оптимизации производительности.

Для решения этих ограничений в Си предусмотрены **встраиваемые ассемблерные вставки** (`inline assembly`), которые позволяют вставлять **машинный код напрямую** в исходную программу. Выглядит он таким образом
```c
asm volatile ( AssemblerTemplate 
                      : OutputOperands
                      : InputOperands
                      : Clobbers
                      : GotoLabels)
```

1. **AssemblerTemplate** 
    - строка с инструкциями ассемблера, где используются заменители(`%0`, `%1` и т.д.) для операндов. 
    - Например `"add %0, %1, %2"` - сложение 1 и 2 операнда с помещением результата в 0 выходной операнд
2. **OutputOperands** 
    - переменные, куда и будет записываться результат выполнения ассемблерного кода. 
    - Формат: `"=ограничение"(название переменной из си)`. 
    - Так например для нашего примера `"add %0, %1, %2"` если мы запишем в OutputOperands `"=r"(result)` - то результат этого сложения запишется в переменную `result`, а ограничение `r` говорит о том, что в ассемблере переменная result будет регистром
3. **InputOperands**
    - Переменные или константы, используемые как входные данные. 
    - Формат: `"ограничения"(переменная)`
    - Для нашего примера `"add %0, %1, %2"`, если мы в InputOperands напишем `"r"a, "r"(b)` то в итоге мы получим что эта операция сделает `result=a+b`
4. **Clobbers**
    - Список регистров или `"memory"` которые модифицируется ассемблерным кодом в **AssemblerTemplate**(естественно кроме тех регистров что уже перечислены в **OutputOperands** и **InputOperands**). 
    - То есть если мы например напишем `"add r0, %1, %2"`, то в таком случае у нас будут только InputOperands, но по факту ведь этот код изменил и r0. А компилятор не будет знать этого если ему явно на это не указать. Поэтому и нужно записать в **Clobbers** `"r0"` чтобы явно сказать компилятору что этот регистр будет изменён. Иначе компилятор будет считать что в r0 находится то же значение что и перед asm volatile
5. **GotoLabels**
    - Использует метки для перехода при помощи goto. Как мы помним goto это фуфуфуфуфу. Но если вдруг в asm используется какая-то инструкция перехода, то вместо `asm volatile` нужно писать `asm goto` а в **AssemblerTemplate** для label писать `%l[label]`
    - Например
    ```c
    `asm goto ("cmp %0, #0; beq %l[error]" 
                :
                : "r"(value) 
                : 
                : error);
    ```

Пример полноценного использования asm volatile
```c
int src = 1;
int dst;   

asm ("mov %1, %0\n\t"
    "add $1, %0"
    : "=r" (dst) 
    : "r" (src));

printf("%d\n", dst);
```

```c
`uint32_t value = 42; uint32_t *port = (uint32_t *)0x40020000; // Записываем значение в аппаратный порт. // Используем "memory", так как изменяем память (аппаратный регистр). 

asm volatile ("str %0, [%1]" 
        :
        : "r"(value), "r"(port) 
        : "memory");
```

## Ограничения для OutputOperands

- `=r` - вывести результат в **регистре**. Компилятор выделит регистр для записи результата.  
    Пример: `: "=r"(dst)` — dst будет записан из регистра.
- `=m` - вывести результат в **память** (адрес работает как целевой).
