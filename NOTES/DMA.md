# Base
Прямой доступ к памяти (Direct memory access - DMA) - механизм внутри микроконтроллера, позволяющий автоматически переносить данные между различными областями по событию. Преимущество DMA в том, что он работает полностью автоматически из-за чего нет необходимости следить за определенными флагами и состояниями внутри программы.

DMA может переносить данные в 4 направлениях:
- из памяти в память, например из одной переменной в другую;
- из памяти в периферию - например для отправки сформированного сообщения по какой-либо из периферий;
- из периферии в память - например для сохранения значений, полученных периферией;
- из периферии в периферию - например для отправки в интерфейс связи данных, полученных другой периферией.

В микроконтроллерах STM32G4 DMA состоит из двух частей:
- непосредственно контроллера прямого доступа к памяти, один из каналов которого мы и настраивали в примере;
- мультиплексор каналов DMA который нужен для того, чтобы иметь возможность подключать любой канал DMA, к любой возможной периферии. Например в STM32G4 доступно 16 каналов DMA поровну разделенные между DMA1 и DMA2 и 115 различных мест, где DMA могут применяться.

## DMA_MUX и DMA
Каналы мультипликатора напрямую подключены к каналам DMA: 
- DMA_MUX каналы с 0 до 7 - DMA1 каналы с 1 по 8;
- DMA_MUX каналы с 8 по 15 - DMA2 каналы с 1 по 8.
# Блок схема DMA
![[Pasted image 20240318185716.png]]
`DMAMUX` (DMA multiplexer) - это модуль мультиплексора DMA, который используется в микроконтроллерах для маршрутизации запросов DMA от различных периферийных устройств к соответствующим каналам DMA.
`DMAMUX` позволяет использовать несколько периферийных устройств с одним и тем же каналом DMA, не создавая конфликтов и помех при передаче данных. DMAMUX также позволяет настроить приоритет запросов DMA от различных устройств, чтобы обеспечить правильную работу системы.

Соответственно `DMAMUX` разбивает одинаковые устройства по двум DMA: `DMA1` и `DMA2`. 

`Arbiter` используется если сразу многие устройства пытаются получить доступ к DMA, то есть он просто расставляет приоритеты для этого и нужен мультиплексор.
 
 `Ch 1` - `Ch 8` это каналы для работы с DMA. 

# DMA перевод
Когда происходит одиночный перевод через DMA происходит следующее:
1. Периферия отправляет сигнал запроса DMA в контроллер DMA.
2. Контроллер DMA обслуживает запрос в зависимости от приоритета канала, связанного с этим запросом периферии.
3. Как только контроллер DMA предоставляет доступ периферии, он отправляет подтверждение периферии.
4. Периферия снимает свой запрос, как только получает подтверждение от контроллера DMA.
5. После того как периферия снимает свой запрос, контроллер DMA снимает подтверждение.

# Пример настройки DMA
После настройки из примера ниже, данные от АЦП будут сохраняться в массиве, на который указывает array_to_write_to каждый раз, когда будут считываться без каких-либо дополнительных действий со стороны программы.
```c
void setup_adc_dma( uint16_t *array_to_write_to )
{
    // Включить тактирование DMA
    RCC->AHB1ENR |= RCC_AHB1ENR_DMA1EN | RCC_AHB1ENR_DMAMUX1EN;

    // Настроить адрес откуда будет происходить чтение
    DMA1_Channel1->CPAR = (uint32_t)&(ADC2->DR);

    // Настроить адрес куда будет происходить записью надо напрямую преобразовать 
    // адрес в число типа uin32_t, так как все адреса - 32 значения.
    DMA1_Channel1->CMAR = (uint32_t)array_to_write_to;

    // Настроить количество значений, которые мы хотим считать.
    DMA1_Channel1->CNDTR = 4;

    // Настроить канал DMA: размер данных 16 бит источник и приемник, циклический 
    // режим, увеличивать указатель при записи.
    DMA1_Channel1->CCR |= 1 << DMA_CCR_MSIZE_Pos | 1 << DMA_CCR_PSIZE_Pos 
                       | DMA_CCR_MINC | DMA_CCR_CIRC;        

    // Настроить мультиплексор каналов DMA
    DMAMUX1_Channel0->CCR |= (36 << DMAMUX_CxCR_DMAREQ_ID_Pos);

    // Включить DMA
    DMA1_Channel1->CCR |= DMA_CCR_EN;
}
```

Теперь для данного канала нам нужно настроить адрес из которого мы берём данные(в нашем случае `ADC2->DR`) и адресс куда мы будем передавать данные(в нашем случае в массив `array_to_write_to`). <span  style="color: red">ВАЖНО! Адреса нужно переводить в 32 битный тип uint32_t пушто dma использует 32 битные адреса.</span> `DMA_CPAR` - это регистр, содержащий адрес периферийного регистра, из которого или в который будут передаваться данные. `DMA_CMAR` - это регистр, содержащий адрес ячейки памяти, из которой или в которую будут передаваться данные. Этот адрес может меняться во время работы DMA-контроллера, что позволяет передавать данные в разные области памяти.
```c
// Настроить адрес откуда будет происходить чтение
DMA1_Channel1->CPAR = (uint32_t)&(ADC2->DR);

// Настроить адрес куда будет происходить запись. надо напрямую преобразовать 
// адрес в число типа uin32_t, так как все адреса - 32 значения.
DMA1_Channel1->CMAR = (uint32_t)array_to_write_to;
```
 Далее мы выбираем количество пакетов которые мы хотим считать
 ```c
 // Настроить количество значений, которые мы хотим считать.
DMA1_Channel1->CNDTR = 4;
```
Далее мы конфигурируем работу DMA при помощи регистра DDR. 
```c
// Настроить канал DMA: размер данных 16 бит источник и приемник, циклический 
// режим, увеличивать указатель при записи.
DMA1_Channel1->CCR |= 1 << DMA_CCR_MSIZE_Pos | 1 << DMA_CCR_PSIZE_Pos 
				   | DMA_CCR_MINC | DMA_CCR_CIRC;        
```

Биты `PSIZE` отвечают за размер пакета который мы будем передавать С ПЕРИФЕРИИ В ПАМЯТЬ

`MINC` (Memory INCREment mode) - это бит конфигурации канала DMA, который отвечает за режим инкремента адреса памяти при передаче данных по каналу DMA.
Когда бит `MINC` установлен в 1, адрес памяти, указанный в регистре `DMA_CMAR` (Current Memory Address Register), автоматически увеличивается после каждой передачи данных по каналу DMA. Это позволяет передавать данные из последовательных ячеек памяти без необходимости изменять адрес памяти вручную.

Когда бит `CIRC` установлен в 1, канал DMA работает в циклическом режиме. Это означает, что после передачи указанного количества данных (заданного в регистре `DMA_CNDTR` - Current Number of Data Register) канал DMA автоматически возвращается к начальному адресу памяти и начинает передачу данных с начала. Это позволяет передавать данные из буфера памяти в периферийное устройство или из периферийного устройства в буфер памяти непрерывно, без необходимости повторной настройки канала DMA.

Далее нам нужно разобраться в DMAMUX с каким устройством мы работаем, это нужно для того чтобы микроконтроллер понимал с чем мы имеем дело, именно для этого нужен DMAMUX потому что до появления DMAMUX просто были заранее установленные устройства для каналов. ВООООТ. ID для устройства находится в главе DMAMUX в Reference manual
