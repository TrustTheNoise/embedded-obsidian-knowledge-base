# Дискретизация
Любой физический сигнал в нашем мире является непрерывным(звук, изменение тока). Благодаря физическим датчикам мы можем получить эти сигналы в виде электрического сигнала. Однако наши компьютеры не могут содержать непрерывный сигнал, это займёт просто гигантское количество памяти да и смысла будет не много. По сему в наших компьютерах обычно находится дискретные отсчёты самого нужного нам сигнала.
То есть допустим у нас есть микрофон который даёт нам непрерывный сигнал, а наш компьютер может хранить только дискретные отсчёты. Поэтому нам нужно создать эквивалентный дискретный сигнал. И вот создание эквивалентного непрерывному сигналу дискретный сигнал и есть дискретизация. Для этого обычно используются Аналого-цифровые преобразователи, но о них можно прочитать в соответствующей заметке(там для stm32g4). Здесь же мы рассмотрим как это происходит математически
## По простому

Вот как это выглядит графически. У нас есть непрерывный сигнал x(t) в котором нам нужно брать значение каждые Ts секунд. Тогда у нас вместо непрерывного сигнала будут дискретные отсчёты, значения которые мы взяли с непрерывного сигнала. И вот как раз таки эти отсчёты и будут являться дискретным эквивалентом для нашего сигнала. Так мы получили дискретный сигнал.
А математически это можно записать так:
У нас есть непрерывный сигнал x(t)
Мы берём в нём отсчёты каждые Ts секунд(что называется периодом дискретизации)
Таким образом у нас будет некоторое количество n отсчётов, причём n - целые числа
То есть мы будем брать значение в момент t=0, Ts, 2Ts, 3Ts, ... , nTs
Таким образом у нас t=nTs, тогда
$$x (t) xrightarrow (t = n T s) x (n T s)=|text ( "но для простоты мы записываем" ) | = x[n]$$
Мы убрали nTs и заменили на $x[n]$ по той причине что у нас $T_(s)$ не меняется и нам важны только целое число отвечающая за номер отсчёта.
## Строгое математическое объяснение
Для того чтобы мы отбирали дискретные отсчёты значений в непрерывной функции мы можем использовать [дельта функцию](Дельта%20функция)
Особенно если ты посмотришь на [[Дельта функция#Как применяется дельта функция]] то можешь сам это понять, её можно легко использовать для отбирания отсчётов.
Поэтому нам нужно умножить наш сигнал на дельта функции которые сдвинуты на nTs т.е.
$$x_(s) (t)=x (t) dot.op sum_(n=-infinity)^(infinity) delta (t-n T_(s))=sum_(n=-infinity)^(infinity) underbrace(x(n T_(s)), x[n]) delta (t-n T_(s))$$

# Квантование
Помощь: https://www.youtube.com/watch?v=yAz0VucxTt8
Квантование — это процедура, в которой непрерывное по амплитуде значение каждого дискретного отсчёта заменяется ближайшим уровнем из конечного набора уровней АЦП.
![[Pasted image 20240429224344.png]]

АЦП не может хранить произвольные значения напряжения, потому что его выход закодирован конечным количеством бит. Если АЦП имеет разрядность N, он может выдавать только $2^(N)$ уровней, равномерно расположенных между 0 и $V_(r e f)$. Больше про этот момент написано в [[АЦП#Base]]]

## Ошибка квантования
При квантовании любой отсчёт цифрового сигнала может содержать ошибку, которая возникает из-за того, что значение сигнала может находиться между двумя соседними уровнями квантования. Эта ошибка называется квантовой ошибкой.

Максимальное значение квантовой ошибки равно половине значения младшего значащего бита («расстояния» между соседними уровнями квантования). Младший значащий бит - это самый правый бит бинарного кода, который соответствует наименьшему значению уровня квантования.

Например, если мы квантуем сигнал с шагом 0,1 В, то младший значащий бит будет равен 0,1 В. Максимальное значение квантовой ошибки в этом случае будет равно ± 0,05 В, то есть половине значения младшего значащего бита.

Эта ошибка возникает из-за того, что при квантовании мы заменяем непрерывное значение сигнала ближайшим дискретным значением, которое соответствует уровню квантования. Эта замена может привести к тому, что значение сигнала будет немного отличаться от истинного значения.
