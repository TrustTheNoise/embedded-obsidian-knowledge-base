# Директории

Одним из базовых понятий в CMake являются **директории**.  
CMake-директория отличается от обычной тем, что в ней есть файл **CMakeLists.txt**. Наличие этого файла означает, что директория участвует в конфигурации сборки.

Каждый `CMakeLists.txt` описывает роль своей директории:
- может объявлять библиотеки или исполняемые файлы,
- может настраивать параметры сборки,
- может просто подключать другие CMake-директории через `add_subdirectory()`.

## Области видимости в CMake

У CMake есть несколько разных механизмов области видимости, и они ведут себя по-разному.
1. Переменные
    Переменные имеют директорную область видимости.
    - При вызове add_subdirectory() значения переменных копируются из родительской директории в дочернюю.
    - Изменения переменных в дочерней директории не влияют на значения в родительской.
    - Область видимости иерархическая, но не двусторонняя.
> [!tldr] 
> дочерняя директория видит значения переменных родителя, но родитель не видит изменений дочерней. 
2. Таргеты
    Таргеты находятся в **глобальной области видимости проекта**.
    Это значит что если в области видимости родительской директории есть какой-то таргет, то дочерняя директория может использовать эти таргеты.
- Свойства принадлежат только конкретной директории и не могут использоваться дочерними директориями
