# Модель OSI
Стандартная модель сетевой коммуникации — это модель ISO, которая определяет 7 уровней, начиная от физической среды и заканчивая интерфейсом приложения.

В embedded большинство коммуникационных интерфейсов обычно реализуют только функциональность уровня 1 (физический уровень). Физический уровень определяет физическое соединение между узлами. То есть например в [[SPI]] мы должны подключить обычные провода которые передают либо логическую 1, либо логический 0. В SPI пакет можно назвать прям кадром из канального уровня OSI с натяжкой, т.к. в SPI есть только начальный бит и стоп бит, остальное это просто обычные биты. 

В [[I2C]] уже есть кроме самих битов данных есть также ID устройства и тот же арбитраж. Которые как раз таки определяются в канальном уровне OSI.
В той же CAN шине тоже есть как определения для физического уровня(то как должен работать трансивер, используется дифференциальный сигнал), а также канальный уровень(обнаружение ошибок через CRC, ACK бит, арбитраж, кадр данных CAN). Функциональность вышележащих уровней обычно реализуется уже программно. Протокольные стандарты, реализующие эти уровни или их части, называются "протоколами верхних уровней".

Вообще следует отметить, что не все уровни реализуются для приложений встроенных сетей. Например, добавление накладных расходов для маршрутизации сообщений на большие расстояния не имеет особого смысла, если сеть не имеет функций "дальней связи".

Кроме того, традиционные реализации 7 уровней требовали бы интерфейса между любыми двумя соседними уровнями, что привело бы к накладным расходам, неприемлемым для встроенных приложений. Поэтому те-же протоколы верхнего уровня для CAN реализуют только выбранную функциональность из верхних уровней, чтобы минимизировать накладные расходы.

- Физический уровень

Передаёт сырые биты по физической среде. То есть описывает каким образом передаются биты(просто логический 1/0, дифференциальный сигнал), ну и соответственно может описывать устройства которые должны передавать эти биты(мол строение трансивера)

- Канальный уровень

Обеспечивает надёжную передачу кадров (фреймов) между двумя узлами, обнаружение ошибок. То есть он описывает как мы будем различать начало/конец пакета, как будем обнаруживать повреждённые данные(CRC, [bit stuffing из CAN](CAN%20шина#Дополняющие%20биты%20(bit%20stuffing)))

- Сетевой уровень

Включает концепции адресации назначения и маршрутизации. Мол стандартизирует то каким образом происходит взаимодействия с хостом. То есть он стандартизирует все ID-шники для работы с хостом.

- Сеансовый уровень 

Позволяет различным хостам в сети начинать и заканчивать сеансы связи.
Управляет токенами: только сторона, имеющая токен, может выполнять критические функции, такие как запись в общую запись базы данных.
Синхронизация: может использоваться для больших передач данных — поддерживает возобновление прерванной передачи

- Уровень представления

Обрабатывает представление данных и кодирует данные стандартизированным способом. То есть если есть стандартизированный словарь объектов для каждого узла. Или например определён список типов данных которые используются в сети.
Также уровень определяет сжатие данных и шифрование

- Прикладной уровень

А это просто прикладные программы, использующие сеть.
