# База
UART (произносится как "ю-арт") является периферийным устройством последовательного ввода-вывода без пересылки тактового сигнала(то есть он является асинхронным интерфейсом). Вместо этого системы должны заранее договориться о скорости передачи данных, и каждая из них должна локально генерировать свой собственный тактовый сигнал. UART используется в таких протоколах, как [[RS-232]] и [[RS-485]].

На Рис.  показана асинхронная последовательная связь.DTE посылает данные в DCE по линии TX и получает данные обратно по линии RX.
![[Pasted image 20240318000909.png]]
 Здесь мы отправляем для начала старт бит(опускаем бит), затем отправляем 7 битов и подаём 1 как стоп бит. В зависимости от настройки микроконтроллера и принимающего устройства может быть также и 8 и 9 бит данных, также может быть дополнительный бит чётности(проверка).
 Типичные скорости передачи данных включают 300, 1200, 2400, 9600, 14400, 19200, 38400, 57600 и 115200 бод. Обычно берутся 9600 бод если скорость не важно либо 115200 если скорость супер нужна.

# Блок схема UART
![[Pasted image 20240317221911.png]]
*диаграма uart*
Тут многое связанно с [[RS-232]], и [[RS-485]], так что об этом всём будет в их документах, здесь тока то что важно нам для работы с UART.
Итак у нас есть начальные `IRQ Interface`, `DMA Interface`, `COM Controller`
В COM Controller у нас находятся все регистры для работы с интерфейсом. `USARTx->TDR` нужен для передачи по TX, а `USARTx->RDR` для чтения, `TxFIFO` используется если используется режим FIFO, и соответсно нужен если нужна очередь.
`DMA Interface`, `IRQ Interface` для работы с DMA и прерываниями соответсвенно.
`Baudrate generator & oversampling` отвечает за деление частоты чтобы она соответствовала внешнему устройству. Как говорилось в [[UART#База]] UART является асинхронным интерфейсом, поэтому он не передаёт внешней периферии тактовую частоту(но вот USART это делает >;3), вместо этого он вместе с внешней периферией договариваются на нужную частоту. Вот чтобы у нас UART работал на нужной частоте, мы берём частоту микроконтроллера и замедляем через `Baudrate generator & oversampling`
`TX Shift Reg` используется для хранения данных которые мы сейчас будет отправлять. То есть `shif register` собирает данные из USART_RDR что мы хотим отправить и затем по битно сдвигом(по тактовым сигналам) отправляет данные через TX. Точно также но только принимает по битно сдвигом принимает данные через RX `RX Shift Reg`
Для примера рассмотрим как это выглядит на `TX Shift Reg`
![[Гиф 8.gif]]
Как можно заметить по гифке, первым shift register отправляет именно LSB. Поэтому по умолчанию UART отправляет первым LSB(но можно настроить и на MSB)`
# UART символы
Длина слова может быть установлена на 7, 8 или 9 бит, путем программирования битов M в регистре USART_CR1
- Длина символа 7 бит: M[1:0] = ‘10’
- Длина символа 8 бит: M[1:0] = ‘00’
- Длина символа 9 бит: M[1:0] = ‘01’

> [!warning] 
>  В режиме длины данных 7 бит не поддерживаются режим Smartcard, режим LIN master и автоматическая скорость передачи (обнаружение кадров 0x7F и 0x55).

![[Pasted image 20240318002141.png]]
*связь регистра M и Stop на то как будут отправляться данные*

По умолчанию сигнал (TX или RX) находится в низком состоянии во время стартового бита. Он находится в высоком состоянии во время стопового бита(ну и соответственно во время того пока не идёт передача/приём).

Символ Idle интерпретируется как целый кадр из "1"
Символ Break интерпретируется при получении "0" в течение периода кадра. В конце кадра break передатчик вставляет 2 стоповых бита.

# UART FIFO и порог
В UART есть отдельные FIFO для передачи(TXFIFO) и приёма(RXFIFO). FIFO включается через установку FIFOEN в USART_CR1 регистре. 
> [!warning] 
>  FIFO работает только в UART, SPI, Smartcard modes

Так как максимальный размер пакета UART(то есть полезной информации без start/stop bit) 9 бит, то длина данных TXFIFO 9 бит, но длина данных RXFIFO 12 бит, т.к. RXFIFO хранит не только сам пакет, но и флаги ошибок связанные с пакетом(Parity error, Noise error и Framing error flags). Но хоть RXFIFO и хранит флаги ошибок, но в USART_RDR хранит только пакет данных 8-9 битов, а флаги доступны через регистр USART_ISR.
> [!important] 
>  Размер пакета TXFIFO 9 бит, а RXFIFO 12 бит, но при этом оба этих FIFO имеют глубину 8 ячеек.

Для TXFIFO/RXFIFO есть флаг TXFT/RXFT который подымается если уровень для FIFO дошёл до порога который выставляется в TXFTCFG и RXFTCFG регистра USART_CR3. Причём
-  RXFT поднимается(и вызывается если включено прерывание) когда в USART_RDR есть 1 пакет данных, а в RXFIFO находится (RXFTCFG-1) данных. Например если мы установим RXFTCFG=0b101, что является порогом когда RXFIFO заполнен полностью
  ![[Pasted image 20250310110717.png]]
  В таком случае RXFT устанавливается когда в FIFO будет FIFO_length-1 пакетов, а в USART_RDR будет 1 пакет. Это нужно для того чтобы при следующем приёме у нас сходу не вызывался overrun flag, так как у нас есть 1 место в RXFIFO
- TXFT флаг поднимается(и вызывается если включено прерывание) если количество пустых ячеек TXFIFO достигает порог выставленный в TXFTCFG битах.
# Порядок настройки UART

<span style="color: cyan">Чтобы передать символ, выполните последовательность действий ниже:</span>
1. Запрограммируйте биты M в регистре USART_CR1 для определения длины слова.
2. Выберите необходимую скорость обмена с помощью регистра USART_BRR.
3. Запрограммируйте количество стоповых битов в регистре USART_CR2.
4. Включите USART, установив бит UE в регистре USART_CR1 в 1.
5. Выберите разрешение DMA (DMAT) в регистре USART_CR3, если должна происходить многобуферная передача. Настройте регистр DMA, как описано в разделе 37.5.10: многопроцессорная передача USART.
6. Установите бит TE в регистре USART_CR1, чтобы отправить пустой фрейм в качестве первой передачи.
7. Запишите данные для передачи в регистр USART_TDR. Повторите это для каждого передаваемого данных в случае однобуферной передачи.
    - Когда режим FIFO отключен, запись данных в регистр USART_TDR сбрасывает флаг TXE.
    - Когда режим FIFO включен, запись данных в регистр USART_TDR добавляет один байт данных в TXFIFO. Записи в регистр USART_TDR выполняются, когда установлен флаг TXFNF. Этот флаг остается установленным, пока TXFIFO не будет заполнен.
8. После записи последнего данного в регистр USART_TDR, дождитесь, пока TC = 1.
    - Когда режим FIFO отключен, это означает, что передача последнего фрейма завершена.
    - Когда режим FIFO включен, это означает, что TXFIFO и сдвиговый регистр пусты. Эта проверка необходима для предотвращения повреждения последней передачи при отключении USART или входе в режим ожидания.

<span style="color: cyan">Процедура приема символа:</span>
1. Запрограммируйте биты M в регистре USART_CR1, чтобы определить длину слова.
2. Выберите желаемую скорость обмена с помощью регистра скорости обмена USART_BRR.
3. Запрограммируйте количество стоповых битов в регистре USART_CR2.
4. Включите USART, установив бит UE в регистре USART_CR1 в '1'.
5. Выберите разрешение DMA (DMAR) в регистре USART_CR3, если должна происходить многобуферная передача. Настройте регистр DMA, как описано в разделе 37.5.10: многопроцессорная передача USART.
6. Установите бит RE в регистре USART_CR1. Это включает приемник, который начинает поиск стартового бита.
Когда символ принимается:
- Когда режим FIFO отключен, бит RXNE устанавливается, чтобы указать, что содержимое сдвигового регистра передано в RDR. Другими словами, данные были получены и могут быть прочитаны (а также их связанные флаги ошибок).
- Когда режим FIFO включен, бит RXFNE устанавливается, чтобы указать, что RXFIFO не пуст. Чтение USART_RDR возвращает самые старые данные, введенные в RXFIFO.
- Генерируется прерывание, если установлен бит RXNEIE (RXFNEIE при включенном режиме FIFO).
- Флаги ошибок могут быть установлены, если во время приема был обнаружен фреймовый сбой, шум, четность или переполнение.
- В режиме многобуферной передачи:
    - Когда режим FIFO отключен, флаг RXNE устанавливается после приема каждого байта. Он сбрасывается при чтении DMA из регистра приемных данных.
    - Когда режим FIFO включен, флаг RXFNE устанавливается, когда RXFIFO не пуст. После каждого запроса DMA из RXFIFO извлекается данные. Запрос DMA активируется, когда RXFIFO не пуст, т.е. есть данные для чтения из RXFIFO.
- В режиме одиночного буфера:
    - Когда режим FIFO отключен, сброс флага RXNE выполняется путем программного чтения из регистра USART_RDR. Флаг RXNE также может быть сброшен путем программирования бита RXFRQ в '1' в регистре USART_RQR. Флаг RXNE должен быть сброшен до окончания приема следующего символа, чтобы избежать ошибки переполнения.
    - Когда режим FIFO включен, флаг RXFNE устанавливается, когда RXFIFO не пуст. После каждой операции чтения из USART_RDR из RXFIFO извлекается данные. Когда RXFIFO пуст, флаг RXFNE сбрасывается. Флаг RXFNE также может быть сброшен путем программирования бита RXFRQ в '1' в регистре USART_RQR. Когда RXFIFO заполнен, первое значение в RXFIFO должно быть прочитано до окончания приема следующего символа, чтобы избежать ошибки переполнения. Флаг RXFNE генерирует прерывание, если установлен бит RXFNEIE. Альтернативно, прерывания могут быть сгенерированы путем программирования бита RXFRQ в '1' в регистре USART_RQR.

# Пример
```c
void setup_gpio(void){
	// включаем тактирование для GPIOB где находятся ножки для USART
	RCC->AHB2ENR |= RCC_AHB2ENR_GPIOBEN;

	// Настраиваем PB3, PB4 на альтернативную функцию
	GPIOB->MODER &= ~(GPIO_MODER_MODE3_Msk | GPIO_MODER_MODE4_Msk);
	GPIOB->MODER |= 2 << GPIO_MODER_MODE3_Pos | 2 << GPIO_MODER_MODE4_Pos;

	// Включаем максимальную скорость PB3, PB4
	GPIOB->OSPEEDR |= 0b11 << GPIO_OSPEEDR_OSPEED3_Pos | 0b11 << GPIO_OSPEEDR_OSPEED4_Pos;

	// включаем для PB3 и PB4 7 альтернативную функцию, потому что именно она
	// отвечает за USART2_TX и USART2_RX соответсвенно(смотри таблицу ниже)
	GPIOB->AFR[0] |= 7 << GPIO_AFRL_AFSEL3_Pos | 7 << GPIO_AFRL_AFSEL4_Pos;
}

void setup_uart(void){
	// включаем тактирование для USART2
	RCC->APB1ENR1 |= RCC_APB1ENR1_USART2EN;

	// Сбрасываем регистр конфигурации CR1
	// Тут как бы и настраивается количество битов данных, но т.к. у нас 8 бит
	// оставляем M0 = 0, M1 = 0
	USART2->CR1 = 0;

	// Выставляем деление частоты на 1661 16000000(16 МГц тактовая частота микроконтроллера
	// по умолчанию)/1661 ~ 9600
	USART2->BRR = 1661;

	// Делаем только 1 стоп бит, расценка ниже
	USART2->CR2 |= ~(USART_CR2_STOP_Msk);

	// Включаем передачу и приём соответсвенно 
	USART2->CR1 |= USART_CR1_TE | USART_CR1_RE;
	// Включаем USART2
	USART2->CR1 |= USART_CR1_UE;
}
```

```c
void put_char_serial(char c){
	// передаём на устройство символ char
	USART2->TDR = c;
	// дожидаемся пока usart2 не передаст сообщение
	while(!(USART2->ISR & USART_ISR_TC)) {}
}

char get_char_serial(void){
	// дожидаемся пока USART2 не примет данные
	while( !(USART2->ISR & USART_ISR_RXNE) ) {}
	// возвращаем принятые данные
	return USART2->RDR;
}
```
