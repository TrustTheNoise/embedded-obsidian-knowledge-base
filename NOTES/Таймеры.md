# Теория
В любом микроконтроллере есть таймер - это периферия, которая способная выполнять счётные операции на основании тактового сигнала(импульсы амплитуда которых изменяется от напряжения питания до нуля)

Таймеры обновляют значения на регистрах и т.д.


# Таймеры
Все таймеры в STM32 могут работать в довольно большом количестве различных режимов и делятся на несколько категорий:
- Продвинутые таймеры (advanced timers: TIM1, TIM8, TIM20);
- Таймеры общего назначения (general purpose timers: TIM2/TIM3/TIM4/TIM5)
- Простые таймеры (Basic timers TIM6/TIM7).
Все таймеры могут работать в базовых режимах счетчика: возрастающего, убывающего, с прерыванием и так далее. Таймеры общего назначения имеют по 4 канала, выведенные на различные ножки микроконтроллера. Данные ножки могут быть использованы как выходы таймера, например для ШИМ, или как входы, например для считывания частоты или показаний инкрементального энкодера.

Продвинутые таймеры имеют по 4 канала, которые могут подключаться сразу к 8 выходам микроконтроллера, например в качестве выходов ШИМ с противофазой и deadtime.

# Time-base unit
Основная часть программируемого таймера это 16/32 битный счётчик с связанным с ним auto-reload register. Счетчик может считать вниз, вверх или "как вверх, так и вниз".

Time-base unit каждого таймера состоит из:
- Регистр счётчика(TIMx_CNT)
- Регистр предделитель(TIMx_PSC)
- Регистр автоперезагрузки(TIMx_ARR)

Регистр автоперезагрузки нужен для того чтобы после достижения счётчика определённого значения наш счётчик сбрасывался и начинал счёт заново. То есть регистр автоперезагрузки показывает также до какого момента будет идти сам счётчик. Допустим если TIMx_ARR=999, то счётчик сделает 1000 отсчётов перед тем, чтобы закончить отсчёты либо начать отсчёты заново с установкой флага переполнения.

Регистр автоперезагрузки предварительно заполнен. Что это значит? Запись в регистр автоперезагрузки или чтение из него осуществляется через теневой регистр предзагрузки. То есть если мы захотим засунуть новое значение в регистр автоперезагрузки, то нам нужно сначала засунуть значение в теневой регистр. И он обновит значение регистра автоперезагрузки после событии обновления(UEV), в зависимости от бита разрешения автоперезагрузки (ARPE) в регистре **TIMx_CR1**.

Это событие вызывается когда происходит переполнение счётчика(ну или мы наоборот дошли до 0 в счётчике на уменьшение) и при этом бит **UDIS**(Update disable) равен 0 в регистре **TIMx_CR1**.

Событие обновления может быть также вызвано программно(поставив UG=1 в регистре TIMx_EGR) 

> [!important] 
> Из-за того что регистр автоперезагрузки использует теневой регистр из которого идёт в последствии значения в регистр автоперезагрузки после  события обновления, то мы можем менять регистр автоперезагрузки на лету. Нам не обязательно останавливать таймер, у нас заменится регистр автоперезагрузки после события обновления. 

## Предделитель 
Счётчик тактируется выходом предделителя, который включается только тогда, когда установлен бит разрешения счетчика (**CEN**) в регистре **TIMx_CR1**
> [!attention] 
> Обратите внимание, что фактический сигнал разрешения счетчика **CNT_EN** устанавливается через 1 такт после установки **CEN**.

Предделитель может разделить встречную тактовую частоту с шины к которой подключен таймер на любой коэффициент от 1 до 65536. Он меняется через 16 битный регистр TIMx_PSC. 
> [!important] 
> Как и регистр автоперезагрузки предделитель тоже можно менять на лету. У нас изменится предделитель после события обновления(UEV)

# Режимы счёта таймера
## Режим прямого счёта(DIR=0)
В **режиме прямого счета** счетчик считает от 0 до значения автоперезагрузки (содержимого регистра **TIMx_ARR**), затем перезапускается с 0 и генерирует событие переполнения счетчика.

Событие обновления (Update Event) может быть сгенерировано при каждом переполнении счетчика или путем установки бита **UG** в регистре **TIMx_EGR** (программно или с помощью контроллера режима подчинения).

Можно отключить событие обновлений если поставить флаг UDIS=1, это нужно для того чтобы регистры TIMx_PSC, TIMx_ARR не обновлялись из регистров предзагрузки. То есть в таком случае после переполнения счётчик перезапускается с 0, генерируется событие переполнения счётчика, но не происходит событие обновления. 
Кроме того, если установлен бит **URS** (выбор запроса обновления) в регистре **TIMx_CR1**, установка бита **UG** генерирует событие обновления **UEV**, но без установки флага **UIF**, который отвечает за прерывание и запрос DMA.
В целом URS нужен для выбора откуда будет вызываться флаг UIF. Если стоит 0 то от всех источников: переполнения, программного вызова UG и через slave контроллер. А если URS=1 то UIF вызывается только от переполнения счётчика.

## Режим обратного счёта(DIR=1)
В режиме обратного счёта счетчик считает от значения автоперезагрузки (содержимого регистра **TIMx_ARR**) до 0, затем перезапускается с значения автоперезагрузки и генерирует событие переподсчета(далее underflow) счетчика.

В остальном тут работает точно также как и в [[#Режим прямого счёта(DIR=0)]] единственное UEV вызывается из-за события underflow. При этом URS, UDIS и т.д. работают также.`

# Пользование таймером
Каждый таймер является независимой периферией, которую, необходимо отдельно тактировать.
Таймеры получают тактовую частоту от шины, на которой находятся. В таблице 3 STM32G4 Series memory map and peripheral register boundary addresses в reference manual написано, что таймеры 2-4 тактируются от шины APB1. То есть входная частота таймера будет равна частоте на этой шине. Для примеров ниже будем считать, что все таймеры 16-битные (хотя на самом деле TIM2 - 32-битный)
![[Pasted image 20231215223910.png]]
Каждый таймер имеет предделитель - счетчик, который выдает на выход сигнал только тогда, когда доходит до заданного значения, после чего обнуляется. Это необходимо для того, чтобы увеличить диапазон возможных значений, с которыми может работать таймер.
Важно понимать, что таймеры считают частоту с которой работают - частоты шины, при работе в режиме базового счетчика, и частоту внешнего источника при работе, например, в режиме считывания инкрементального энкодера. В нашем случае счетчик будет считать такты на шине APB1.

Текущее состояние таймера можно прочитать в `TIMx_CNT` - счетном регистре.
Предделитель настраивается в регистре  `TIMx_PSC`. При этом в регистр надо записывать значение на 1 меньше, чем желаемая величина предделителя, так как счет начинается с 0.

При переполнении таймера происходит событие. Таймер можно настроить таким образом, чтобы событие останавливало таймер, сбрасывало значение счетного регистра в 0 и автоматически запускало таймер снова. Кроме этого, можно настроить вызов прерывания по переполнению.

Настройка верхнего предела, при котором произойдет переполнение, то есть значения, до которого будет заполнять счетный регистр при счете в положительную сторону и с которого будет начинаться счет при счете в отрицательную, производится в регистре `TIMx_ARR`. Как и в случае с предделителем, при записи необходимо отнимать 1 от желаемого значения.
##  Примеры
При максимальной настройке предделителя таймер можно считать 32-битным без возможности получить значение первых 16 бит, так как и предделитель и счетный регистр таймера являются 16-битными.

Текущее состояние таймера можно прочитать в TIMx_CNT - счетном регистре.

Предделитель настраивается в регистре `TIMx_PSC`. При этом в регистр надо записывать значение на 1 меньше, чем желаемая величина предделителя, так как счет начинается с 0.

При переполнении таймера происходит событие. Таймер можно настроить таким образом, чтобы событие останавливало таймер, сбрасывало значение счетного регистра в 0 и автоматически запускало таймер снова. Кроме этого, можно настроить вызов прерывания по переполнению. Это очень удобно для реализации логики, завязанной на реальное время, например для реализации более точной задержки.

Настройка верхнего предела, при котором произойдет переполнение, то есть значения, до которого будет заполнять счетный регистр при счете в положительную сторону и с которого будет начинаться счет при счете в отрицательную, производится в регистре `TIMx_ARR`. Как и в случае с предделителем, при записи необходимо отнимать 1 от желаемого значения.

Допустим, что `SYSCLK` и `APB1` настроены на частоту 32 МГц и мы хотим получить прерывание ровно 2 раза в секунду чтобы переключать светодиод. При переключении с частотой 2 Гц получится, что 0.5 секунд светодиод включен, а другие 0.5 выключен.

При максимальной настройке предделителя таймер можно считать 32-битным без возможности получить значение первых 16 бит, так как и предделитель и счетный регистр таймера являются 16-битными.

Текущее состояние таймера можно прочитать в TIMx_CNT - счетном регистре.

Процесс вычисления коэффициентов предделителя может выглядеть следующим образом.

32 МГц это 32000000 Гц - тактов в секунду, соответственно, чтобы получить необходимый нам период в 0.5 с, надо сократить частоту до 2 Гц, то есть произведение PSC + 1 на ARR + 1 должно быть равно 32000000/2 = 16000000. Если стоит задача просто получить значение задержки, то совершенно неважно, какие конкретно значения будут в PSC и ARR, главное чтобы их произведение равнялось 16000000. Но при этом удобно выбираться значение PSC так, чтобы ARR был равен периоду в миллисекундах, то есть в нашем случае 499 (500 - 1). Это позволит считывать из TIMx_CNT количество миллисекунд, прошедших с последнего события на таймере, если это будет необходимо. 

Это один из множества возможных подходов к подбору коэффициентов деления, можно придумать и свои подходы, когда они лучше подходят под задачу. Если пользоваться описанной выше логикой, то ARR возьмем 499 (500 - 1), а PSC 16000000/500 - 1 = 31999. Важно следить, чтобы значения, записываемые в регистры, не превышали максимального возможного, для 16-битных регистров 65535 иначе значение задержки будет неточным.

Код запуска таймера 2 для выполнения описанной выше логики показан в примере ниже. При вызове прерывания будем переключать состояние ножки PE8.
```c
#include "stm32g474xx.h"
int main (void)
{
	setup_clock(); // Настройка частоты микроконтроллера
	
	// Включение тактирования
	RCC->AHB2ENR |= RCC_AHB2ENR_GPIOEEN;
	RCC->APB1ENR1 |= RCC_APB1ENR1_TIM2EN;
	
	// Настройка PE8 на выход
	GPIOE->MODER &= ~( GPIO_MODER_MODE8_Msk);
	GPIOE->MODER |= 1 << GPIO_MODER_MODE8_Pos;
	
	// Настройка таймера
	TIM2->PSC = 31999;
	TIM2->ARR = 499;
	TIM2->DIER |= TIM_DIER_UIE; // Разрешение прерывания по переполнению
	TIM2->CR1 |= TIM_CR1_CEN; // Включение таймера
	NVIC_EnableIRQ (TIM2_IRQn); // Включение прерывания в NVIC
	
	while ( 1 )
	{
	
	}
}

// Обработчик прерывания по переполнению таймера
void TIM2_IRQHandler (void)
{
	GPIOE->ODR ^= GPIO_ODR_OD8;
	TIM2->SR &= ~ TIM_SR_UIF; // Сброс прерывания по таймеру
}
```
Таймер может считать как от меньшего к большему, то есть в положительную сторону - от 0 до ARR, так и от большего к меньшему, то есть в отрицательную сторону - от ARR до 0. Для изменения стороны, в которую считает таймер надо установить бит DIR в TIMx_CR1 регистре:
```c
TIM2->CR1 |= TIM_CR1_DIR;
```
