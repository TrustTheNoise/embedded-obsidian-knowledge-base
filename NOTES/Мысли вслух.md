# Прослойка для разных типов функций

Если в си для какого-то интерфейса у нас есть несколько вариантов, допустим профайлинг через stream и через buffer. Можно сделать статическую структуру в .c файле который будет включать в себя основные функции

```c
typedef enum {
    PROFILING_STREAM,
    PROFILING_BUFFER,
} profiling_debug_type;

typedef struct {
    void (*start)(profiling_profile*);
    void (*stop)(profiling_profile*);
} profiling_operations;
```

И соответственно будет два разных setup, один setup будет для stream, а второй для buffer. И соответственно каждый из этих настроек будет в функции start и stop.
