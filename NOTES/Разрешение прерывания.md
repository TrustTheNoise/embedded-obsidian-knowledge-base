## Base
Для того, чтобы разрешить прерывание в NVIC, библиотека CMSIS предоставляет специальную функцию - NVIC_EnableIRQ, и набор определений, которые можно найти в заголовочном файле, подключенном в проект, то есть для нашего случая stm32g474xx.h. Небольшая часть определений показана на рисунке 4.2.4.
![](https://lh7-us.googleusercontent.com/ZCChIk7tV2Cb9XkUfvarvLEaXVHLf6A22igHJYHIPLAfFFTYFtDp0m8jStmYd_k3QhIMslIaShzlMU91Biwx9VjyGWzjYll6s0vV14-AytohKMDmgdws7ChSI3cl6K5c_jnfjnDeT-G-xwCp9f05FA)
Очень важно заметить, что для обработки внешних прерываний 0-4 существуют свои, отдельные обработчики и разрешения, а вот прерывания для ножек с большим номером обрабатываются уже вместе. Внешние прерывания 5-9 обрабатываются с использованием функции EXTI9_5_IRQHandler, а прерывания 10-15 с использованием функции EXTI15_10_IRQHandler. При этом, если включено несколько прерываний одновременно, узнать какое именно из них произошло можно с помощью бит в регистре EXTI_PR1.
Ножка микроконтроллера, которая будет использована в качестве источника прерывания, может быть настроена в любой режим, но логичнее всего настроить ее в режим цифрового входа.
