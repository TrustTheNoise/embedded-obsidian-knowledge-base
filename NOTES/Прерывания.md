# EXC_Return
Когда ты переходишь в прерывание в Cortex-M то регистр LR работает иным образом. Потому что во время обработчика прерываний Cortex-M переходит в другой режим работы

В Cortex-M есть два режима работы:
- Handler mode(режим обработчика) - процессор выполняет обработчик прерывания. В этом режиме всегда используется **MSP (Main Stack Pointer)**.
- Thread mode (потоковый режим) - режим обычного выполнения задач. Потоковый режим может использовать либо **MSP**, либо **PSP (Process Stack Pointer)** в зависимости от конфигурации.

Так вот при входе в обработчик прерываний в регистр LR записывается не какой-то адрес, а специальный код EXC_RETURN, старшие 28 бит которого установлены в 1, а биты [3:0] содержат информацию, необходимую для выполнения операции возврата
![[Pasted image 20251023225112.png]]
- Бит 0 - состояние процессора(ARM/Thumb) после возврата. Для Cortex‑M всегда 1, так как процессор поддерживает только Thumb-режим.
- Бит 2 показывает какой будет использоваться стек при выходе из обработчика прерываний: 1 = MSP (Main Stack Pointer), 0 = PSP (Process Stack Pointer).
- Бит 3 - режим, в который вернётся процессор: 1 = потоковый режим (Thread mode), 0 = обработчик прерывания (Handler mode). Для первого обработчика прерывания этот бит будет установлен, потому что при выходе мы должны перейти в режим потока. Но вот если вызовутся вложенные прерывания, то в них этот бит будет очищен, так как при выходе из вложенного прерывания мы опять входим в прерывание, а значит нам нужен режим прерывания
