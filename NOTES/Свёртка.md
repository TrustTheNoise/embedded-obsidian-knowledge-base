# Дискретная свёртка

Из [свойств линейных систем](Линейно%20инвариантные%20во%20времени%20системы(LTI)) следует фундаментальный результат: если разложить любой сигнал на набор **дельта-импульсов**, пропустить каждый импульс через систему **отдельно**, а затем **сложить все отклики**, то сумма будет **точно совпадать** с откликом системы на исходный сигнал.

Причина проста: линейная система **масштабирует и сдвигает каждый входной импульс**, а отклики на разные импульсы **не взаимодействуют** друг с другом (суперпозиция и однородность). Поэтому реакция системы на сложный сигнал полностью определяется реакциями на его элементарные импульсы.

Это означает, что для анализа системы достаточно знать её **импульсную характеристику** $h[n]$, т.е. отклик на единичную [дельта функцию](Дельта%20функция) $\delta[n]$ то есть на единичный импульс. 
![[Pasted image 20251201194837.png|350]]
Если две любые системы имеют хотя бы какие-то различия, их импульсные характеристики будут обязательно отличаться.

Любой импульсный сигнал, являющийся компонентом разложения при импульсной декомпозиции, может быть представлен как единичный импульс, сдвинутый на соответствующую величину по временной оси и умноженный на значение отсчёта исходного сигнала в этот же момент времени. Для примера рассмотрим сигнал а[п], все значения которого равны нулю, кроме 8-го отсчёта, принимающего значение -3. Этот сигнал может рассматриваться как единичный импульс, сдвинутый (задержанный) на 8 периодов дискретизации и умноженный на -3, или в форме математического выражения: 
$x_2[n] = -3\delta[n-5]$
![[Pasted image 20251201193912.png|300]]

Если на единичный импульс система даёт отклик $h[n]$, то на сигнал $x_2[n] = -3\delta[n-8]$ выход будет:
$y_2[n] = -3h[n-5]$
![[Pasted image 20251201195032.png|800]]
То есть импульсная характеристика сдвигается во времени и масштабируется по амплитуде в точном соответствии с входным сигналом.

Таким образом, зная $h[n]$, можно рассчитать реакцию системы на любой сигнал: разложить его на импульсы, получить отклики на каждый из них и сложить результаты получив отклик системы на изначальный вход.

![[Pasted image 20251201200217.png|750]]

Именно это фундаментальное свойство и легло в основу математической операции, известной как свёртка.

Свёртка — это математическая операция, которая позволяет вычислить отклик линейной системы на любой входной сигнал, зная её отклик на единичный импульс (импульсную характеристику). Она заключается в суммировании откликов системы на все элементарные импульсы, составляющие входной сигнал, с учётом их сдвига во времени и масштаба.

Свёртка в виде суммы каждого отклика на единичный импульс может быть вычислена с помощью кода на языке программирования Си.
```c
uint32_t signal[SIGNAL_SIZE]; // входной сигнал
uint32_t impulse_response[RESPONSE_SIZE]; // импульсная характеристика
uint32_t output_signal[SIGNAL_SIZE + RESPONSE_SIZE - 1]; // выход

void convolution()
{
    for(uint32_t i=0; i < SIGNAL_SIZE; i+=1)
    {
        for(uint32_t j=0; j < RESPONSE_SIZE; j+=1)
        {
            output_signal[i+j] += 
                    signal[i]*impulse_response[j];
        }
    }
}
```

## Математическая формула свёртки

Ранее, в [[#Дискретная свёртка]], мы рассматривали процесс «физически»: каждый отсчёт входного сигнала создаёт целый «след» на выходе через импульсную характеристику. Это удобно интуитивно, но плохо подходит для точной математики. 

Математическая запись свёртки делает наоборот: она выражает один отсчёт выхода через сумму произведений соответствующих отсчётов входного сигнала и импульсной характеристики. 

![[Pasted image 20251206160832.png|450]]
Механизм свёртки со стороны выхода выглядит так. Чтобы получить отсчёт выхода свёртки y[n] мы берём n-M(где M размер импульсной характеристики) последних отсчётов исходного сигнала и умножаем на <span style="color:rgb(255, 0, 0)">зеркально отражённую</span> импульсную характеристику, затем мы суммируем все результаты произведения. При этом мы сдвигаем импульсную характеристику на n отсчётов.

Таким образом формула будет выглядеть так:
$$y[n] = x[n] \ast h[n]=\sum_{m=0}^{M-1}x[n-m]h[m]$$

## Размер выходного сигнала

Выходной сигнал свёртки всегда будет с размером N+M-1, где N - размер входного сигнала. M - размер выходного сигнала.
Представь сигнал x как длинную ленту из N клеток и h — как «окно» длины M, которое сдвигается вдоль ленты и в каждой позиции берётся взвешенная сумма перекрывающихся клеток, выдавая одно значение y[n] для каждой позиции окна, при которой оно хоть немного перекрывает ленту, даже частично, даже на один элемент, — таких позиций ровно N+M−1:

## Краевой эффект

На картинке ниже которая показывает механизм реализации свёртки становится видна одна из важных проблем, возникающих при реализации свёртки
![[Pasted image 20251210104750.png|400]]
На картинке мы вычисляем выходной отсчёт y[0]. При этом в процессе вычислений должны участвовать входные отсчёты x[-3], x[-2], x[-1], однако этих отсчётов в сигнале просто нет! Такая же проблема возникла если бы мы вычисляли например последний элемент выхода y[11], который требует x[9], x[10], x[11] то есть индексы, которые превышают длину входного сигнала.

Решение может состоять в том, чтобы сформировать недостающие входные отсчёты каким-либо образом и дополнить ими сигнал слева и справа. Очевидным шагом является дополнение нулями . При этом программа реализации свёртки вместо того, чтобы обращаться к несуществующим входным элементам, будет загружать нули. Поскольку нулевой элемент, участвуя в операции умножения, не вносит изменений в результат обработки, свёртка, выполненная указанным образом, будет математически эквивалентна свёртке, выполненной с учётом модификации механизма вычислений на краях.

Есть, однако, одно важное замечание. Крайние правые и крайние левые от
счёты выходного сигнала формируются на основе неполной информации. Ведь в них будут участвовать элементы сигналы которые мы дополнили нулями. Из-за чего выходной сигнал в крайних правых и крайних левых отсчётах будет некорректен. На языке ЦОС говорят, что импульсная характеристика не является полностью согласованной с входным сигналом. Если импульсная характеристика включает М отсчётов, то первые и последние (М - 1) отсчёты выходного сигнала будут вычислены на основе меньшего числа входных данных, чем остальные. Здесь можно провести аналогию с перходной характеристикой, ведь нам точно также нужно время для стабилизации, входа в установившийся режим.

На рисунке показан пример того, как описанные краевые эффекты могут приводить к неправильной работе системы. 
![[Pasted image 20251210110208.png|850]]
В данном примере мы при помощи ВЧ фильтра убираем постоянную составляющую. Однако проблема в том, что 30 первых и 30 последних выходных отсчётов неверны из-за дополнения нулями изначального сигнала. Сигнал на выходе получается при обработке всего такого «удлинённого» входного сигнала. Проблема «краевого эффекта» широко распространена в ЦОС. 

> [!important] 
> Можно принять за общее правило, что начальные и конечные отсчёты обрабатываемых сигналов следует игнорировать. 

# Свёртка в частотной области

У [свёртки](Свёртка) имеет два основных недостатка. Во‑первых, математически она требует многократного умножения и сложения, что усложняет обратный анализ — трудно восстановить входной сигнал по выходному. Во‑вторых, свёртка требует большого числа операций, что сильно замедляет вычисления.

Например, если фильтр имеет импульсную характеристику длиной 512 отсчётов, а микроконтроллер работает на 200 МГц с арифметикой с плавающей точкой, формирование одного выходного отсчёта стандартной свёрткой займёт примерно 1 мс. Пропускная способность составит всего 1000 отсчётов в секунду — в 40 раз меньше, чем требуется для качественного аудио, и в 10 000 раз меньше, чем для телевизионного вещания.

Однако есть решение которые выполняет ту же свёртку куда быстрее. Это использование частотной характеристики системы. Мы переводим входной сигнал и импульсную характеристику в спектр с помощью DFT. Тогда спектр выходного сигнала вычисляется просто:

$$|Y[k]| = |X[k]|\ \cdot\ |H[k]|$$
$$\angle Y[k] = \angle X[k] + \angle H[k]|$$

Обратное DFT возвращает выходной сигнал во временной области, эквивалентный линейной свёртке. Хоть и с нюансами.

Во первых использовать обычное DFT, выигрыш в скорости будет незначительным: нужно вычислить DFT входного сигнала и импульсной характеристики, перемножить спектры и сделать обратное DFT. 
Ситуация кардинально изменилась с появлением метода [Быстрого Преобразования Фурье](Быстрое%20преобразование%20Фурье(FFT).md), который сильно укоряет преобразование. Из-за чего свёртка через частотную область получается почти в 100 раз быстрее чем свёртка во временной области.

## Циклическая свёртка

Другой нюанс куда труднее. Проблема в том что как мы помним после свёртки входного сигнала с размером N и импульсной характеристикой с размером M мы получаем на выходе сигнал с размером N+M-1. 
А в случае DFT мы при преобразовании входного сигнала с размером N получаем спектр с отсчётами в диапазоне $[0; N-1]$. 
Поэтому нам нужно привести и входной сигнал и импульсную характеристику к одинаковому количеству отсчётов, чтобы можно было перемножить спектры. Привести к одинаковому количеству отсчётов можно просто дополнив сигнал нулями
![[Pasted image 20251203214712.png|500]]
Проблема в том, что после перемножения спектров и обратного DFT выходной сигнал тоже имеет длину 256, а не 256+51−1. В результате мы получаем «сжатый» сигнал с меньшим числом отсчётов, который к тому же отличается от правильного результата свёртки во временной области.

Причина кроется в природе DFT. Как мы вывели из [[#Вывод DFT через Ряд Фурье]] в случае DFT мы считаем что наш сигнал был размножен бесконечное количество раз, что у нас в итоге получается периодический сигнал. То же самое относится и к импульсной характеристике.

То есть важно понять мы рассматриваем сигналы так, будто они периодические. Соответственно при умножении спектров мы фактически выполняем свёртку **периодических сигналов**.
Каждую свёртку можно условно разбить на один период входного сигнала и один период импульсной характеристики каждый по 256 отсчётов. Но в импульсной характеристике у нас ненулевые только первые 51 отсчёт, так что можем считать что у нас обычная свёртка 256 отсчётов входного сигнала на 51 отсчёт импульсной характеристики

Таким образом после свёртки каждого периода у нас получается сигнал на 256+51-1=306 отсчётов. Однако период DFT составляет только 256 отсчётов. Из-за этого последние 50 отсчётов свёртки «выходят» за пределы периода и накладываются на начало следующего периода из-за чего свёртка получается некорректной в начале.
![[Pasted image 20251203215735.png|750]]

Когда же мы делаем обратное DFT мы убираем периодический сигнал оставляя только один период, только 256 отсчётов. В начале этого периода оказываются наложенные отсчёты, которые пришли из «конца» предыдущего периода. В итоге первые 50 отсчётов выходного сигнала искажаются, а общая длина сигнала остаётся 256 вместо правильных 306 отсчётов. Такой эффект называется циклической свёрткой.

$$F\{x[n]\cdot h[n]\}=X(\Omega) \circledast H(\Omega)=\int_{-\pi}^{\pi}X(\Omega-\Theta)H(\Theta)d\Theta$$

> [!important] 
> Как можно понять циклическая свёртка это один из примеров ситуации, когда важно понимать что DFT рассматривает сигнал как периодический. Хотя об этом очень легко забыть 

Поэтому решением для циклической свёртки будет взятие за количество отсчётов L=N+M-1 или больше, где N - количество отсчётов исходного сигнала, M - количество отсчётов импульсной характеристики. И таким образом нужно будет дополнить нулями как исходный сигнал так и импульсную характеристику до количество отсчётов L, а затем уже выполнять свёртку через частотную область с помощью DFT для L отсчётов.
