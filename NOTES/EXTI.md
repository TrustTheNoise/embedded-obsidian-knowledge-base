## EXTI
Для работы с внешними прерываниями существует специальная периферия - контроллер внешних прерываний и событий - Extended interrupts and events controller (EXTI).
Для того, чтобы включить какое-то прерывание, нам необходимо настроить соответствующую маску в регистре EXTI_IMR1/2, а также выбрать по какому из событий - фронту или спаду сигнала, будет происходить прерывание (регистры EXTI_RTSR1/2 и EXTI_FTSR1/2 соответственно), после чего разрешить прерывание в NVIC.
![[Pasted image 20231022153527.png]]
Кроме этого, после того, как происходит прерывание, в регистрах EXTI_PR1/2 появляется “1” на позиции прерывания, которое произошло. То есть для EXTI4 это будет PIF4 и так далее. Данные флаги нужно обязательно чистить в обработчике прерывания, так как новое прерывание не будет разрешено до момента, пока данный флаг не будет сброшен. Например код разрешения прерывания по фронту для EXTI4 с использованием CMSIS будет выглядеть следующим образом:
```c
EXTI->IMR1 |= EXTI_IMR1_IM13;
EXTI->RTSR1 |= EXTI_RTSR1_RT13;
```
А обработчик может выглядеть так:
```c
void EXTI4_IRQHandler()
{
	// Код который нужно выполнить в прерывании
	EXTI->PR1 = EXTI_PR1_PIF4; // Очистка флага прерывания
}
```
