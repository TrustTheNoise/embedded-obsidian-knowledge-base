# Тактовый сигнал АЦП
Тактовый сигнал АЦП выводится из системного тактового сигнала или из выхода "P" PLL. Выбор тактового сигнала для АЦП производится в RCC_CCIPR в ADCxxSEL
![[Pasted image 20241124125451.png]]

Но это только выбор источника тактового сигнала. Он может быть делен на следующие значения предделителя: 1, 2, 4, 6, 8, 10, 12, 16, 32, 64, 128 или 256 путем настройки регистра ADCx_CCR(это ADC common control register), а именно значения `PRESC[3:0]`. 

Тактирование АЦП в таком случае асинхронна по отношению к тактовому сигналу AHB. 

Альтернативно, тактовый сигнал АЦП может быть выведен из тактового сигнала AHB интерфейса шины АЦП, деленного на программируемый коэффициент (1, 2 или 4). Этот программируемый коэффициент настраивается с использованием полей битов CKMODE в регистре ADCx_CCR.

# Включение/выключение АЦП

По умолчанию АЦП находится в режиме глубокого энергосбережения, где его питание внутренне отключено для уменьшения токов утечки (бит DEEPPWD в регистре ADC_CR по умолчанию установлен в 1).

Когда операции АЦП завершены, АЦП можно отключить (ADEN = 0). Для экономии энергии также можно отключить регулятор напряжения АЦП, установив бит ADVREGEN = 0.

Затем, чтобы еще больше сэкономить энергию, уменьшив токи утечки, можно снова войти в режим глубокого энергосбережения АЦП, установив бит DEEPPWD = 1 в регистре ADC_CR. Это особенно полезно перед входом в режим остановки.

Примечание: Установка DEEPPWD = 1 автоматически отключает регулятор напряжения АЦП, и бит ADVREGEN автоматически сбрасывается.

При отключении регулятора напряжения (ADVREGEN=0) мы не теряем каллибровку АЦП, в отличии от включении режима глубокого энергосбережения(DEEPPWD=1)

## Включение АЦП
1. Выйдите из режима глубокого энергосбережения, сбросив бит DEEPPWD.
2. Включите регулятор напряжения АЦП, установив бит ADVREGEN.
3. Очищаем ADRDY в ADC_ISR регистре записывая туда 1
4. Включаем АЦП(ADEN=1). После этого АЦП нужен время для стабилизации $t_(S T A B)$. Флаг ADRDY будет установлен когда АЦП будет готов к работе
5. Дожидаемся когда ADRDY=1. Это можно сделать через прерывание (установив ADRDYIE=1)
6. Очищаем ADRDY бит через тот же самый пункт 2(это опционально, но тогда придёцца очищать перед следующий включением)

> [!Caution] 
>  ADEN бит не может быть установлен пока идёт каллибровка(ADCAL=1) и в течении 4х циклов после аппаратного сброса ADCAL

> [!caution] 
>  ПОВТОРЯЮ ДЛЯ ТУПЫХ! ADRDY НЕ СБРАСЫВАЕТСЯ САМОСТОЯТЕЛЬНО!

## Выключение АЦП
1. Проверьте что не АЦП не производит преобразование(ADSTART=0 и JADSTART=0). Если необходимо, остановите любое преобразование установив ADSTP = 1 and JADSTP = 1 and then wait until ADSTP = 0 and JADSTP = 0.
2. Установите ADDIS=1
3. Если это требуется приложением, дождитесь, пока ADEN = 0, пока аналоговый АЦП не будет фактически отключен (ADDIS автоматически сбрасывается, когда ADEN = 0).
# Одноконцевой(single-ended) и дифференциальный вход в каналах
## Что это вообще такое
Для начала разберёмся что это и с чем это едят
### single-ended input
В режиме **одноконцевого входа(single-ended input)** АЦП измеряет напряжение относительно фиксированного уровня напряжения, обычно это земля (GND) или отрицательное опорное напряжение (V REF−).
В этом режиме: 
**Положительный вход ($V_(I N P[i])$)**: Это напряжение, которое вы хотите измерить.
**Отрицательный вход**: Фиксированное напряжение, обычно GND или $V_(R E F−)$.
Формула для вычисления входного напряжения в одноконцевом режиме
$$V_(i n)=V_(I N P[i])-V_(R E F−)$$
### дифференциальный вход
В режиме дифференциального входа АЦП измеряет разность между двумя входными напряжениями: положительным ($V_(I N P[i])$) и отрицательным ($V_(I N N[i])$). В этом режиме:

- **Положительный вход ($V_(I N P[i])$)**: Одно из входных напряжений.
- **Отрицательный вход ($V_(I N N[i])$)**: Другое входное напряжение.

Формула для вычисления входного напряжения в дифференциальном режиме:
$$V_(i n)=V_(I N P[i])-V_(I N N[i])$$
# Виды каналов
Есть два вида каналов в АЦП с которых можно преобразовывать данные:
- обычные
- инжектированные
## Обычные каналы
1. **Цель**: Используются для регулярного и периодического преобразования аналоговых сигналов в цифровые.
2. **Инициация**: Могут быть инициированы как программно, так и аппаратно (например, по таймеру или внешнему событию).
3. **Последовательность**: Преобразования выполняются последовательно, один за другим.
4. **Приоритет**: Имеют низкий приоритет по сравнению с инжектированными каналами.
5. **Использование**: Подходят для задач, где требуется регулярное и последовательное преобразование данных, например, для мониторинга температуры или напряжения.

## Инжектированные каналы АЦП
1. **Цель**: Используются для быстрого и приоритетного преобразования аналоговых сигналов в цифровые.
2. **Инициация**: Могут быть инициированы только аппаратно (например, по таймеру или внешнему событию). Программное инициирование возможно только если очередь инжектированных каналов отключена.
3. **Последовательность**: Преобразования выполняются с высоким приоритетом и могут прерывать регулярные преобразования.
4. **Приоритет**: Имеют высокий приоритет и могут прерывать регулярные преобразования для выполнения своих задач.
5. **Использование**: Подходят для задач, где требуется быстрое реагирование на события, например, для обработки аварийных ситуаций или быстрого измерения критических параметров.

# Калибровка АЦП
АЦП предоставляет автоматическую процедуру калибровки, которая управляет всей последовательностью калибровки, включая последовательность включения/выключения питания АЦП. В ходе процедуры АЦП вычисляет калибровочный коэффициент, который имеет ширину 7 бит и применяется внутренне к АЦП до следующего выключения питания АЦП. Во время процедуры калибровки приложение не должно использовать АЦП и должно ждать, пока калибровка не будет завершена.
Калибровка является предварительной для любой операции АЦП.

Калибровочный коэффициент, применяемый для преобразований с одноконцевым входом, отличается от коэффициента, применяемого для преобразований с дифференциальным входом:
- Если поставить ADCALDIF=0 то калибровка будет применяться для преобразований с одноконцевым входом
- Если поставить ADCALDIF=1 то калибровка будет применяться для преобразований с дифференциальным входом.

После этого нужно установить ADCAL=1 для начала калибровки(это нужно делать пока АЦП отключен ADEN=0). Бит ADCAL остается в состоянии 1 на протяжении всей последовательности калибровки. Затем он сбрасывается аппаратно сразу после завершения калибровки. В этот момент соответствующий калибровочный коэффициент сохраняется внутренне в аналоговом АЦП и также в битах `CALFACT_S[6:0]` или `CALFACT_D[6:0]` регистра `ADC_CALFACT` (в зависимости от калибровки для одноконцевого или дифференциального входа).

Внутренняя аналоговая калибровка сохраняется, если АЦП отключен (ADEN = 0). Однако, если АЦП отключен на продолжительное время, рекомендуется выполнить новый цикл калибровки перед повторным включением АЦП.

Внутренняя аналоговая калибровка теряется каждый раз, когда питание АЦП отключается (например, когда устройство переходит в режим ожидания или VBAT). В этом случае, чтобы избежать затрат времени на повторную калибровку АЦП, возможно повторно записать калибровочный коэффициент в регистр ADC_CALFACT без повторной калибровки, предполагая, что программное обеспечение ранее сохранило калибровочный коэффициент, полученный во время предыдущей калибровки.

Калибровочный коэффициент может быть записан, если АЦП включен, но не выполняет преобразование (ADEN = 1 и ADSTART = 0 и JADSTART = 0). Затем, при следующем запуске преобразования, калибровочный коэффициент автоматически вводится в аналоговый АЦП. Эта загрузка происходит прозрачно и не добавляет никакой задержки в циклах к началу преобразования. Рекомендуется перекалибровать, когда напряжение VREF+ изменилось более чем на 10%.

## Процедура простой калибровки АЦП
1. Убедитесь, что DEEPPWD = 0, ADVREGEN = 1 и что время запуска регулятора напряжения АЦП истекло.
2. Убедитесь, что ADEN = 0.
3. Выберите режим входа для этой калибровки, установив ADCALDIF = 0 (одноконцевой вход) или ADCALDIF = 1 (дифференциальный вход).
4. Установите ADCAL.
5. Дождитесь, пока ADCAL = 0.
6. Калибровочный коэффициент можно прочитать из регистра ADC_CALFACT.
7. (Если нужно и single-ended и дифференциальные входы) тогда нужно процедуру 3-6 но с другим ADCALDIF 
## Процедура ввода прошлых данных в ADC_CALFACT
1. Убедитесь, что ADEN = 1, ADSTART = 0 и JADSTART = 0 (АЦП включен и не выполняется преобразование).
2. Запишите CALFACT_S и CALFACT_D с новыми калибровочными коэффициентами.
3. Когда запускается преобразование, калибровочный коэффициент вводится в аналоговый АЦП только в том случае, если внутренний аналоговый калибровочный коэффициент отличается от того, который сохранен в битах CALFACT_S для одноконцевого входного канала или в битах CALFACT_D для дифференциального входного канала.

# Ограничения при записи битов управления АЦП
Записывать биты управления RCC для включения тактового сигнала АЦП, биты управления DIFSEL в ADC_DIFSEL ADCAL ADEN и ADC_CR только пока ADEN=0

Биты управления ADSTART, JADSTART и ADDIS в ADC_CR пока ADEN=1 и ADDIS=0

Для остальных битов в ADC_CFGR, ADC_SMPRx, ADC_TRy, ADC_SQRy, ADC_JDRy, ADC_OFRy, ADC_OFCHRy и ADC_IER:
- Для битов управления, связанных с настройкой регулярных преобразований нужно чтобы ADEN=1 ADSTART=0
- Для битов управления, связанных с настройкой инжектированных преобразований нужно чтобы ADEN=1 JADSTART=0
- Регистры ADC_TRy могут быть изменены, когда выполняется аналого-цифровое преобразование

Программное обеспечение может записывать биты управления ADSTP или JADSTP в регистре ADC_CR только в том случае, если АЦП включен, возможно, выполняет преобразование, и нет ожидающего запроса на его отключение (ADSTART или JADSTART должен быть равен 1, а ADDIS — 0).

# Выбор каналов
В каждом АЦП может быть до 19 мультиплексированных каналов:
- До 13 медленных аналоговых входов, поступающих с контактов GPIO (ADCx_INP/INN[6:18]). В зависимости от продукта, не все из них могут быть доступны на контактах GPIO.
- АЦП подключены к следующим внутренним аналоговым входам:
    - Внутреннее опорное напряжение (VREFINT) подключено к ADC1_INP18, ADC3_INP18, ADC4_INP18 и ADC5_INP18.
    - Внутренний датчик температуры (V TS) подключен к ADC1_INP16 и ADC5_INP4.
    - Канал мониторинга VBAT (VBAT/3) подключен к ADC1_INP17, ADC3_INP17 и ADC5_INP17.

Примечание: Для преобразования одного из внутренних аналоговых каналов соответствующие аналоговые источники должны быть сначала включены путем программирования битов VREFEN, VBATSEL или VSENSESEL в регистрах ADCx_CCR.

## Преобразования в группе
Можно делать группы которые будут преобразовываться по порядку. Например у нас будет такая последовательность преобразований: ADCx_INP/INN3, ADCx_INP/INN8, ADCx_INP/INN2, ADCx_INN/INP2, ADCx_INP/INN0
Есть два вида групп преобразований:
- Регулярная группа может состоять до 16 преобразований. Регулярные каналы и их порядок в последовательности преобразований должны быть выбраны в регистрах ADC_SQRy. Общее количество преобразований в регулярной группе должно быть записано в биты `L[3:0]` в регистре ADC_SQR1.

Пример для последовательной группы.
```c
ADC1->SQR1 = (4 << ADC_SQR1_L_Pos); // 5 преобразований
ADC1->SQR1 |= (3 << ADC_SQR1_SQ1_Pos); // Канал 3 будет 1
ADC1->SQR1 |= (8 << ADC_SQR2_SQ2_Pos); // Канал 8 будет 2
ADC1->SQR1 |= (2 << ADC_SQR2_SQ3_Pos); // Канал 2 будет 3
ADC1->SQR1 |= (2 << ADC_SQR2_SQ4_Pos); // Канал 2 будет 4
ADC1->SQR2 |= (1 << ADC_SQR2_SQ5_Pos); // Канал 1 будет 5
```
Когда мы начнём преобразования то у мы будем делать преобразования в этой последовательности и преобразования будут записаны в ADR_DR
> [!caution] 
> Важно понимать что если мы будем дожидаться преобразования всей последовательности то у нас в ADR_DR будет только последнее преобразование. В нашем случае там будут данные из 1 канала 

> [!warning] 
> ADC_SQRy нельзя менять пока происходят преобразования
> ADC_JSQRy можно изменять на лету, даже когда происходит преобразования

> [!note] 
> Как я понял кроме как делать вот такие группы по другому нельзя делать преобразования. То есть даже если мы захотим сделать всего 1 преобразование то нужно будет делать очередь из 1 канала

# Программируемое время для считывания показаний с АЦП
Перед началом преобразования АЦП нужно установить прямое соединение между измеряемым источником. Смысл в том что ацп заряжает встроенный конденсатор до напряжения, а ему сюрприз-сюрприз нужно время на то чтобы банально зарядится. Поэтому мы можем настроить то время которое мы будем давать этому конденсатору на зарядку.
Для этого мы используем биты `SMPx[2:0]` в ADC_SMPR1-ADC_SMPR2 где x в SMPx это номер канала
• SMP = 000: 2.5 ADC clock cycles
• SMP = 001: 6.5 ADC clock cycles
• SMP = 010: 12.5 ADC clock cycles
• SMP = 011: 24.5 ADC clock cycles
• SMP = 100: 47.5 ADC clock cycles
• SMP = 101: 92.5 ADC clock cycles
• SMP = 110: 247.5 ADC clock cycles
• SMP = 111: 640.5 ADC clock cycles
Т.е. если у нас частота для ADC - 30 МГц и мы сделали для 1 канала SMP1=000, а время которое нам нужно для полного преобразования равно

$$T_(C O N V)=S M P+1 2.5 " АЦП тактов"$$
В таком случае для преобразование данных с 1 канала
$$T_(C O N V)=(2.5+1 2.5) " АЦП тактов" = 1 5 " АЦП тактов" = 1 5*frac(1, 3 0 М Г ц)=5 0 0 n s$$
> [!warning] 
> Нам нужно обязательно расчитывать так чтобы у нас было время выборки не менее возможной для АЦП(такая информация обычно говорится в даташите). Например <span style="color:red">для stm32g474 минимальное время для выборки $frac(2.5, f_(A D C))$ а максимальное $frac(6 4 0.5, f_(A D C))$ *stm32g474x datasheet 3.18* </span>

> [!tip] 
>  АЦП также уведомляет о конце фазы выборки, устанавливая статусный бит EOSMP (только для обычных преобразований). Флаг EOSMP сбрасывается программным обеспечением путем записи 1 в него. Прерывание может быть сгенерировано, если установлен бит EOSMPIE.

## Режим одиночного преобразования (CONT=0)
В режиме одиночного преобразования после того как мы начнём преобразовывать данные ((J)ADSTART=1) АЦП будет ПОЛНОСТЬЮ ПРИНИМАТЬ ВСЕ КАНАЛЫ В ПОСЛЕДОВАТЕЛЬНОСТИ, а потом остановится пока мы снова не сделаем ADSTART=1.
Когда у нас произойдёт 1 преобразование в последовательности, то у нас поднимется флаг (J)EOC ну и если мы поставим (J)EOCIE то вызовется прерывание
Когда у нас пройдёт преобразование всей последовательности то у нас поднимется флаг (J)EOS ну и если мы поставим (J)EOSIE то вызовется прерывание

(J) - это для непрерывных каналов
## Режим непрерывного преобразования (CONT=1)
Тут всё то же самое, те же флаги будут подниматься. Единственное когда у нас закончится преобразование всей последовательности то у нас АВТОМАТИЧЕСКИ НАЧНЁТСЯ ОПЯТЬ ПРЕОБРАЗОВАНИЕ ВСЕЙ ПОСЛЕДОВАТЕЛЬНОСТИ
## Режим прерывистого преобразования
### Регулярных групп(DISCEN=1)
В этом режиме мы можем преобразовывать короткую последовательность(подгруппу) из n преобразований (n ≤ 8), которая является частью последовательности преобразований, выбранной в регистрах ADC_SQRy. Значение n задается записью в биты `DISCNUM[2:0]` регистра ADC_CFGR.
Т.е. у нас есть полная последовательность которую мы расписали в ADC_SQRy, но если у нас стоит DISCEN=1 то мы будем за один раз преобразовывать только n каналов из последовательности а затем останавливаться. При этом флаг EOC и EOS будет точно также подниматься как и в других режимах, EOC после каждого преобразования, EOS после преобразования всей последовательности
Пример: DISCEN=2(n=3), каналы в последовательности = 1, 2, 3, 6, 7, 8, 9, 10, 11
– 1-й триггер: преобразованные каналы 1, 2, 3. 
– 2-й триггер: преобразованные каналы 6, 7, 8 (событие EOC генерируется на каждом преобразовании). 
– 3-й триггер: преобразованные каналы 9, 10, 11 (событие EOC генерируется на каждом преобразовании) и событие EOS генерируется после преобразования канала 11. 
– 4-й триггер: преобразованные каналы 1, 2, 3 (событие EOC генерируется на каждом преобразовании).
### Инжектированных групп(JDISCEN=1)
Здесь то же самое что и для DISCEN=1, только естественно для инжектированных каналов, а также здесь нельзя выбрать сколько будет каналов в подгруппе что мы будем преобразовывать за раз. Т.е. здесь нет своего аналога DISNUM. Поэтому когда мы включаем JDISCEN, у нас в любом случае будет n=1. Поэтому когда будет происходить триггер мы будем преобразовывать только 1 канал в последовательности ADC_JSQR, а затем останавливаться преобразование
Пример:
JDISCEN = 1, каналы для преобразования = 1, 2, 3
- 1-й триггер: преобразуется канал 1 (генерируется событие JEOC)
- 2-й триггер: преобразуется канал 2 (генерируется событие JEOC)
- 3-й триггер: преобразуется канал 3 и генерируются события JEOC и JEOS
- 4-й триггер: преобразуется канал 1 (генерируется событие JEOC)
- ...

> [!warning] 
> Невозможно использовать одновременно [режим автоинжекции](#Режим%20автоинжекции(JAUTO=1)) и прерывистый режим: биты DISCEN и JDISCEN должны быть очищены программным обеспечением, когда установлен бит JAUTO.

## Само начало преобразований(ADSTART, JADSTART)
Если у нас `(J)EXTEN[1:0]` равен 0, то сразу после (J)ADSTART=1 у нас начнётся преобразование последовательности в [режиме что мы выбрали](АЦП.md#Режимы%20преобразования%20в%20группе). Ежели у нас `(J)EXTEN[1:0]` не равен 0, тогда после запуска (J)ADSTART=1 у нас начнётся преобразования только после вызова триггера что мы выбрали в (J)EXTSEL

> [!note] 
> В режиме автоинжекции (JAUTO = 1) используйте бит ADSTART для запуска регулярных преобразований, за которыми следуют автоинжектированные преобразования (бит JADSTART должен оставаться сброшенным). 

Биты ADSTART и JADSTART также предоставляют информацию о том, выполняется ли в данный момент какая-либо операция АЦП.

ADSTART сбрасывается аппаратно:

- В одиночном режиме с программным регулярным триггером (CONT = 0, EXTSEL = 0x0). При любом завершении последовательности регулярного преобразования (установка EOS) или при любом завершении обработки подгруппы, если DISCEN = 1
- Во всех случаях (CONT=x, EXTSEL=x). После выполнения процедуры ADSTP, утвержденной программным обеспечением.

> [!note] 
> В непрерывном режиме (CONT = 1) ADSTART не сбрасывается аппаратно при утверждении EOS, потому что последовательность автоматически перезапускается. 

JADSTART сбрасывается аппаратно:

- В одиночном режиме с программным инжектированным триггером (JEXTSEL = 0x0). При любом завершении последовательности инжектированного преобразования (установка JEOS) или при любом завершении обработки подгруппы, если JDISCEN = 1
- Во всех случаях (JEXTSEL=x). После выполнения процедуры JADSTP, утвержденной программным обеспечением.

## Остановка преобразований(ADSTP, JADSTP)
Можно останавливать ацп во время преобразований данных. Для этого нужно поставить (J)ADSTP для остановки регулярных(инжектированных) преобразований. Остановка преобразований сбрасывает текущую операцию АЦП. Затем АЦП можно перенастроить (например, изменить выбор канала или триггер) для новой операции.
> [!tip] 
> Можно остановить инжектированные преобразования, пока регулярные преобразования все еще выполняются, и наоборот. Это позволяет, например, перенастроить последовательность инжектированных преобразований и триггеры, пока регулярные преобразования все еще выполняются (и наоборот).

Когда мы вызываем (J)ADSTP любое текущее регулярное(инжектированное) преобразование прерывается и (J)ADC_DR не обновляется. Последовательность сканирования также прерывается и сбрасывается (что означает, что повторный запуск АЦП начнет новую последовательность).

После вызова (J)ADSTP биты (J)ADSTART сбрасываются автоматически. И программа должно опрашивать (J)ADSTART, пока бит не будет сброшен, прежде чем предполагать, что АЦП полностью остановлен.

> [!warning] 
>  В режиме автоинжекции (JAUTO = 1) установка бита ADSTP прерывает как регулярные, так и инжектированные преобразования (бит JADSTP не должен использоваться).

## Управление инжектированными каналами
### Режим триггерной инжекции(JAUTO=0)
Для использования триггерной инжекции бит JAUTO в регистре ADC_CFGR должен быть очищен.
1. Запустите преобразование группы обычных каналов либо по внешнему триггеру, либо установив бит ADSTART в регистре ADC_CR.
2. Если происходит внешний инжектированный триггер или устанавливается бит JADSTART в регистре ADC_CR во время преобразования группы обычных каналов, текущее преобразование сбрасывается, и запускается последовательность инжектированных каналов (все инжектированные каналы преобразуются один раз).
3. Затем возобновляется обычное преобразование группы обычных каналов с того места, где оно было прервано.
4. Если обычное событие происходит во время инжектированного преобразования, инжектированное преобразование не прерывается, но обычная последовательность выполняется после завершения инжектированной последовательности.
### Режим автоинжекции(JAUTO=1)
Если бит JAUTO в регистре ADC_CFGR установлен, то каналы в инжектированной группе автоматически преобразуются после группы обычных каналов. Это может использоваться для преобразования последовательности до 20 преобразований, запрограммированных в регистрах ADC_SQRy и ADC_JSQR.

В этом режиме бит ADSTART в регистре ADC_CR должен быть установлен для запуска обычных преобразований, за которыми следуют инжектированные преобразования (бит JADSTART должен оставаться очищенным). Установка бита ADSTP прерывает как обычные, так и инжектированные преобразования (бит JADSTP не должен использоваться).

В этом режиме внешний триггер на инжектированных каналах должен быть отключен.

Если бит CONT также установлен вместе с битом JAUTO, обычные каналы, за которыми следуют инжектированные каналы, преобразуются непрерывно.
> [!warning] 
> Нельзя использовать одновременно режим JAUTO и [прерывистый режим](#Режим%20прерывистого%20преобразования) 

Когда DMA используется для экспорта данных обычного секвенсора в режиме JAUTO, необходимо запрограммировать его в циклическом режиме (установить бит CIRC в регистре DMA_CCRx). Если бит CIRC сброшен (режим однократного выполнения), последовательность JAUTO останавливается при событии завершения передачи DMA.

> [!help] 
>  Я задавался вопросом будет ли преобразования с инжектированных каналов записываться в ADC_DR вместе с регулярными каналами. Мол если у нас всегда при JAUTO=1 будет инжектированные каналы идти после регулярных то было бы удобно всё хранить в DMA. Но к сожалению это не так. Главный смысл JAUTO только в том чтобы мы запускали все преобразования только по одному триггеру ADSTART

## Режим отложенного преобразования(AUTDLY=1)
Бит AUTDLY (Automatic Delay) в АЦП (аналого-цифровом преобразователе) используется для управления задержкой между последовательными преобразованиями. Вот основные функции и поведение, связанные с битом AUTDLY:

1. **Задержка между преобразованиями**:
Когда бит AUTDLY установлен (AUTDLY = 1), АЦП автоматически вводит задержку между последовательными преобразованиями. Эта задержка необходима для обеспечения стабильности и точности преобразований, особенно в режимах, где требуется синхронизация между несколькими АЦП или между различными последовательностями преобразований.
2. **Игнорирование новых триггеров**:
В режиме AUTDLY любые новые события триггера, происходящие во время текущей последовательности преобразований и связанной фазы задержки, игнорируются. Это предотвращает конфликты и обеспечивает, что каждое преобразование завершается корректно перед началом следующего.
3. **Синхронизация с другими режимами**:
Бит AUTDLY может быть использован в комбинации с другими режимами, такими как одновременный режим (DUAL mode) или режим внедренных преобразований. В этих режимах AUTDLY помогает обеспечить, что все АЦП работают синхронно и что данные преобразований не теряются и не искажаются из-за неправильной синхронизации.

## Размер данных и выравнивание
По завершении каждого обычного преобразования канала (когда происходит событие EOC), результат преобразованных данных сохраняется в регистре данных ADC_DR, который имеет ширину 16 бит.

По завершении каждого инжектированного преобразования канала (когда происходит событие JEOC), результат преобразованных данных сохраняется в соответствующем регистре данных ADC_JDRy, который также имеет ширину 16 бит.

## Смещение
Смещение для канала y (y = 1, 2, 3, 4) может быть применено к каналу путем установки бита OFFSETy_EN = 1 в регистре ADC_OFRy. Канал, к которому применяется смещение, программируется в битах OFFSETy_CH[4:0] регистра ADC_OFRy. В этом случае преобразованное значение в ADC_DR уменьшается на пользовательское смещение, записанное в битах OFFSETy[11:0]. Т.е. в конечном счёте в ADC_DR будет
$$A D C_D R=A D C_D R-O F F S E T$$
Результат может быть отрицательным значением, поэтому считываемые данные являются знаковыми, и бит SEXT представляет расширенное значение знака.

**Примечание**: Коррекция смещения не поддерживается в режиме передискретизации. Когда установлены биты ROVSE и/или JOVSE, значение бита OFFSETy_EN в регистре ADC_OFRy игнорируется (рассматривается как сброшенное).

## Переполнение АЦП(OVR, OVRMOD)
Флаг переполнения (OVR) уведомляет о том, что произошло событие переполнения буфера, когда данные обычного преобразования не были прочитаны (ЦП или DMA) до того, как стали доступны новые преобразованные данные.

Флаг OVR устанавливается, если флаг EOC все еще равен 1 в момент завершения нового преобразования. Прерывание может быть сгенерировано, если бит OVRIE = 1. Когда происходит переполнение, АЦП продолжает работать и может продолжать преобразование, если программное обеспечение не решит остановить и сбросить последовательность, установив бит ADSTP = 1. Флаг OVR сбрасывается программным обеспечением путем записи 1 в него.

Возможно настроить, сохраняются ли данные или перезаписываются при возникновении события переполнения, программируя управляющий бит OVRMOD:

- **OVRMOD = 0**: Событие переполнения сохраняет данные в регистре от переполнения: старые данные сохраняются, а новые преобразования отбрасываются и теряются.
- **OVRMOD = 1**: Регистр данных перезаписывается последним результатом преобразования, и предыдущие непрочитанные данные теряются.

## И наконец мы управляем последовательностью преобразований
### Без DMA
#### С OVR
Если преобразования достаточно медленные, последовательность преобразований может быть обработана программным обеспечением. В этом случае программное обеспечение должно использовать флаг EOC (End of Conversion) и его связанное прерывание для обработки каждого данного. Каждый раз, когда преобразование завершается, устанавливается EOC, и регистр ADC_DR может быть прочитан. OVRMOD должен быть настроен на 0 для обработки событий переполнения как ошибки.
#### Без OVR
Иногда полезно позволить АЦП преобразовывать один или несколько каналов без чтения данных каждый раз (например, если используется аналоговый сторожевой таймер). В этом случае бит OVRMOD должен быть настроен на 1, и флаг OVR должен быть игнорирован программным обеспечением. Событие переполнения не препятствует АЦП продолжать преобразование, и регистр ADC_DR всегда содержит последние преобразованные данные.
### С DMA
Поскольку преобразованные значения каналов сохраняются в единственном регистре данных, полезно использовать DMA для преобразования более одного канала. Это предотвращает потерю данных, уже сохраненных в регистре ADC_DR.

Когда режим DMA включен (установлен бит DMAEN в регистре ADC_CFGR в режиме одиночного АЦП или MDMA отличен от 00 в режиме двойного АЦП), запрос DMA генерируется после каждого преобразования канала. Это позволяет передавать преобразованные данные из регистра ADC_DR в место назначения, выбранное программным обеспечением.

Однако, если происходит переполнение (OVR = 1), потому что DMA не смог обслужить запрос DMA передачи вовремя, АЦП прекращает генерировать запросы DMA, и данные, соответствующие новому преобразованию, не передаются DMA. Это означает, что все данные, переданные в ОЗУ, могут считаться достоверными.

В зависимости от конфигурации бита OVRMOD, данные либо сохраняются, либо перезаписываются
Запросы DMA передачи блокируются до тех пор, пока программное обеспечение не сбросит бит OVR.

Два разных режима DMA предлагаются в зависимости от использования приложения и настраиваются битом DMACFG регистра ADC_CFGR в режиме одиночного АЦП или битом DMACFG регистра ADC_CCR в режиме двойного АЦП:
- **Режим однократного DMA (DMACFG = 0)**: Этот режим подходит, когда DMA настроен на передачу фиксированного количества данных.
- **Циклический режим DMA (DMACFG = 1)**: Этот режим подходит, когда DMA настроен в циклическом режиме.

## Режим автоматической задержки преобразования(AUTDLY)
Автоматическая задержка преобразований полезна для упрощения программного обеспечения, а также для оптимизации производительности приложения, работающего на низкой частоте, где существует риск переполнения АЦП.
Когда AUTDLY = 1, новое преобразование может начаться только после того, как все предыдущие данные той же группы были обработаны: 
- Для регулярного преобразования: после того, как регистр ADC_DR был прочитан или если бит EOC был очищен. 
- Для внедренного преобразования: когда бит JEOS был очищен (см. Рисунок 121). 

Это способ автоматически адаптирует скорость АЦП к скорости системы, которая читает данные.

Задержка вставляется после каждого регулярного преобразования (независимо от значения DISCEN = 0 или 1) и после каждой последовательности инжектированных преобразований (независимо от значения JDISCEN = 0 или 1). <span style="color:red">То есть задержка в инжектированной последовательности будет происходить только после преобразования ВСЕЙ ПОСЛЕДОВАТЕЛЬНОСТИ</span>

Во время задержки внешние триггеры(инжектированные или регулярные) игнорируются.
> [!important] 
> Однако это не относится к программным триггерам, где во время этой задержки остается возможность установить биты ADSTART или JADSTART для перезапуска преобразования: программное обеспечение должно читать данные перед запуском нового преобразования. Тут всё таки сам пользователь решает когда перезапустить а не какой-нибудь таймер...

> [!important] 
> Задержка не вставляется между преобразованиями разных групп (регулярное преобразование, за которым следует внедренное преобразование, или наоборот): 
> - Если внедренный триггер происходит во время автоматической задержки регулярного преобразования, внедренное преобразование начинается немедленно (см. Рисунок 121). 
> - После завершения внедренной последовательности АЦП ожидает задержки (если она не закончена) предыдущего регулярного преобразования перед запуском нового регулярного преобразования (см. Рисунок 123).
> > [!warning] 
> Однако в режиме автоматического внедрения(JAUTO=1) этот процесс отличается. Новое регулярное преобразование может начаться только после завершения автоматической задержки предыдущей внедренной последовательности преобразований (когда JEOS был очищен). Это гарантирует, что программное обеспечение может прочитать все данные определенной последовательности перед началом новой последовательности
> Для остановки преобразования в режиме непрерывного автоматического внедрения, комбинированного с режимом автоматической задержки (JAUTO = 1, CONT = 1 и AUTDLY = 1), следуйте следующей процедуре:

1. Подождите, пока JEOS = 1 (больше не запускаются преобразования).
2. Очистите JEOS.
3. Установите ADSTP.
4. Прочитайте регулярные данные.

# Режим передискретизации(oversampler)
Блок передискретизации выполняет предварительную обработку данных, чтобы разгрузить ЦП. Он способен обрабатывать несколько преобразований и усреднять их в одно значение с увеличенной шириной данных, до 16 бит. Он предоставляет результат в следующей форме, где N и M могут быть настроены:
$$R e s u l t=frac(1, 2^(M)) sum^(n=N-1)_(n=0) C o n v e r s i o n (t_(n))$$
Он позволяет выполнять следующие функции аппаратно: усреднение, уменьшение скорости передачи данных, улучшение отношения сигнал/шум, базовую фильтрацию.
Коэффициент передискретизации N определяется с использованием битов `OVFS[2:0]` в регистре ADC_CFGR2 и может варьироваться от 2x до 256x. Коэффициент деления M состоит из правого битового сдвига до 8 бит и определяется с использованием битов `OVSS[3:0]` в регистре ADC_CFGR2.
> [!important] 
> Главное что тебе нужно понять, так это то что если у нас будет допустим 2 регулярных канала в АЦП, при этом N=2(OVFS=0), тогда у нас каждый канал будет преобразовываться по 2 раза и усредняться

После того как мы сделаем N преобразований одного канала мы делим полученное значение на $2^(M)$, то есть сдвигаем вправо на M битов. После этого мы берём 16 битов и передаём их в ADC_DR. Однако важно понимать, что у нас может быть OVSS=0, то есть никакого сдвига не будет. Однако мы всё равно будем обрезать 20 битные полученные данные до 16 битов, все старшие биты будут сброшены. Также после сдвига выполняется округление с использованием младших значащих битов(т.е. будет происходит математическое округление 10.5=11). То есть только после всего вот этого мы будем предавать в ADC_DR.
> [!help] 
>  Возможно ты спросишь. Как же мне понять какой размер конвертированных данных с АЦП будет в итоге если я использую оверсэплер?
>  Итак чтобы это понять нам для начала нужно узнать какое разрешение в АЦП выставлено без оверсэмплера(ну то есть что в RES). Пусть это будет 12 битов дефолтных.
>  Затем зная количество сэмплов N и сдвиг M мы и можем узнать конечный размер. Если в идеальном случае то максимальное значение конвертированных данных без оверсэмпла это $2^(1 2)-1=4 0 9 5$. И в идеальном случае мы просто будем получать одинаковые 4095 в течении N сэмплов, а в конце просто будем делить $2^(M)$. Но прикол в том что если ты посмотришь OVSR которое и отвечает за количество сэмплов, то поймёшь что там всё это степень 2. Значит можно сказать что мы умножаем на $N=2^(x)$ и делим на $2^(M)$. Да конечно это в идеальном случае, так как в каждом сэмпле может быть разное значение. Но мы рассматриваем максимальный случай.
>  Соответственно чтобы высчитать конечное количество битов в конверсиях АЦП мы можем посчитать формулу
>  $$R E S+log_(2) ( frac(N, 2^(M)) )$$

Преобразование с передискретизацией будет выполняться за время 
$$t=N dot.op T_(C O N V)=N dot.op (t_(S M P L)+t_(S A R))$$
Ну то есть как я и говорил теперь для преобразования 1 канала теперь нужно N раз преобразовать этот канал.

Флаги устанавливаются следующим образом:
- Флаг окончания фазы выборки (EOSMP) устанавливается после каждой фазы выборки.
- Флаг окончания преобразования (EOC) возникает один раз за N преобразований, когда передискретизированный результат доступен.
- Флаг окончания последовательности (EOS) возникает после завершения последовательности передискретизированных данных (то есть после N x длины последовательности преобразований в общей сложности).

## Режимы АЦП поддерживаемые в режиме передискретизации
В режиме передискретизации поддерживаются большинство режимов работы АЦП:

- [Одиночные или непрерывные преобразования.](#Режимы%20преобразования%20в%20группе)
- [Преобразования АЦП запускаются либо программно, либо с помощью триггеров.](#Преобразование%20по%20внешнему%20триггеру(EXTSEL,%20EXTEN,%20JEXTSEL,%20JEXTEN))
- Остановка АЦП во время преобразования (прерывание).
- Чтение данных через ЦП или DMA с обнаружением переполнения.
- Низкоэнергопотребляющие режимы (AUTDLY).
- [Программируемое разрешение](#Размер%20данных%20и%20выравнивание): в этом случае уменьшенные значения преобразований (в соответствии с битами `RES[1:0]` в регистре ADC_CFGR1) накапливаются, обрезаются, округляются и сдвигаются таким же образом, как и 12-битные преобразования.

> [!help] 
> Нужно объяснить насчёт разрешения. То есть мы выполняем передискретизацию значений с разрешением которое мы выставили в RES, но вот в итоговом виде мы всё равно получаем 16 битное число, как и при обычной передискретизации. Т.е. если в RES мы поставили 6 бит, то у нас будет происходить сумма N 6 битных преобразований канала. НО итоговый вариант с сдвигом и всем таким итого будет 16 битным!  

> [!warning] 
> [Режим выравнивания](#Размер%20данных%20и%20выравнивание) недоступен при работе с передискретизированными данными. Бит ALIGN в регистре ADC_CFGR1 игнорируется, и данные всегда предоставляются с выравниванием вправо. 
> [Коррекция смещения](#Компенсация%20смещения) не поддерживается в режиме передискретизации. Когда установлены биты ROVSE и/или JOVSE, значение бита OFFSETy_EN в регистре ADC_OFRy игнорируется (считается сброшенным).

### watchdog и передискретизатор
Функциональность аналогового сторожевого таймера сохраняется (биты AWDSGL и AWDEN), с следующими различиями:

- Биты `RES[1:0]` игнорируются, сравнение всегда выполняется с использованием полных 12-битных значений `HT[11:0]` и `LT[11:0]`.
- Сравнение выполняется по 12 старшим битам 16-битных передискретизированных результатов `ADC_DR[15:4]`.

> [!warning] 
> Следует быть осторожным при использовании высоких значений сдвига, так как это уменьшает диапазон сравнения. Например, если передискретизированный результат сдвинут на 4 бита, что дает 12-битные данные с выравниванием вправо, эффективное сравнение аналогового сторожевого таймера может быть выполнено только по 8 битам. Сравнение выполняется между `ADC_DR[11:4]` и `HT[0:7] / LT[0:7]`, и `HT[11:8] / LT[11:8]` должны быть сброшены. 

### Простейший фильтр(триггерный режим)
Передискретизатор также может использоваться для базовой фильтрации, так как он усредняет значения. Хотя это не очень мощный фильтр (медленное снижение и ограниченное подавление в полосе заграждения), он может использоваться как режекторный фильтр для отклонения постоянных паразитных частот (обычно исходящих от сети или от импульсного блока питания). Для этой цели может быть включен специальный дискретный режим с битом TROVS в регистре ADC_CFGR2, чтобы можно было иметь частоту передискретизации, определяемую пользователем и независимую от самого времени преобразования.

- В возобновляемом режиме(ROVSM=1) накопление возобновляется с 0 (предыдущие результаты преобразований игнорируются). Опять же по примеру

### Передискретизация с режимом двойного АЦП
Возможно включить передискретизацию при работе в конфигурации двойного АЦП для режимов одновременной инжекции и одновременного регулярного режима. В этом случае оба АЦП должны быть запрограммированы с одинаковыми настройками (включая передискретизацию).

Все остальные режимы двойного АЦП не поддерживаются при включенной передискретизации регулярных или инжектированных каналов (ROVSE = 1 или JOVSE = 1).
# Двойное АЦП(21.4.30)
**Dual ADC Mode** (режим двойного АЦП) позволяет использовать два или более АЦП в одном устройстве для одновременного или поочередного преобразования аналоговых сигналов. В режиме двойного АЦП начало преобразования инициируется поочередно или одновременно АЦП-мастером для АЦП-слейва, в зависимости от выбранного режима, определяемого битами DUAL[4:0] в регистре ADCx_CCR.

Реализованы четыре возможных режима:
- Режим одновременного инжекционного преобразования
- Режим одновременного регулярного преобразования
- Режим чередования
- Режим поочередного триггера

Также возможно использование этих режимов в следующих комбинациях:
- Режим одновременного инжекционного преобразования + Режим одновременного регулярного преобразования
- Режим одновременного регулярного преобразования + Режим поочередного триггера
- Режим одновременного инжекционного преобразования + Режим чередования

В режиме двойного АЦП (когда биты `DUAL[4:0]` в регистре ADCx_CCR не равны нулю), биты CONT, AUTDLY, DISCEN, `DISCNUM[2:0]`, JDISCEN, JQM, JAUTO регистра ADC_CFGR совместно используются мастером и слейвом АЦП: биты в слейве АЦП всегда равны соответствующим битам мастера АЦП.
Также биты выбора триггера(`EXTEN[1:0]`, EXTSEL, `JEXTEN[1:0]`, JEXTSEL) устанавливаются только в мастер АЦП.

В режимах одновременного регулярного преобразования или чередования: как только пользователь устанавливает бит ADSTART или бит ADSTP мастера АЦП, соответствующий бит слейва АЦП также автоматически устанавливается. Однако бит ADSTART или бит ADSTP слейва АЦП не обязательно сбрасывается одновременно с битом мастера АЦП.

В режимах одновременного инжекционного преобразования или поочередного триггера: как только пользователь устанавливает бит JADSTART или бит JADSTP мастера АЦП, соответствующий бит слейва АЦП также автоматически устанавливается. Однако бит JADSTART или бит JADSTP слейва АЦП не обязательно сбрасывается одновременно с битом мастера АЦП.

В режиме двойного АЦП преобразованные данные мастера и слейва АЦП могут быть прочитаны параллельно, путем чтения общего регистра данных АЦП (ADCx_CDR). Биты состояния также могут быть прочитаны параллельно путем чтения регистра состояния двойного режима (ADCx_CSR).

## Режим одновременных инжектированных преобразований(`DUAL[4:0]=0b00101`)
Этот режим преобразует инжектированную группу каналов. То есть регулярные каналы на двух АЦП могут дальше быть в разнобой преобразованы. [Внешний источник триггера](#Преобразование%20по%20внешнему%20триггеру(EXTSEL,%20EXTEN,%20JEXTSEL,%20JEXTEN)) поступает от мультиплексора внедренной группы мастер-АЦП (выбранного битами JEXTSEL в регистре ADC_JSQR).

Когда вызывается триггер инжектированного канала, то инжектированная последовательность обоих АЦП начинает преобразовываться одновременно. Если каналов на обоих последовательностях для АЦП мастера и АЦП слейва, тогда в конце преобразования будет устанавливаться событие JEOS только в мастере АЦП. Если же каналов в последовательностях не одинаковое количество, тада у каждого АЦП будет вызываться в своё время событие JEOS(логично)

Этот режим может работать вместе с [JDISCEN=1](#Инжектированных%20групп(JDISCEN=1)). А уже в свою очередь режим может работать с [режимом автоматической задержки преобразования(AUTDLY)](#Режим%20автоматической%20задержки%20преобразования(AUTDLY)) чтобы преобразования точно работали вместе.

- После завершения одновременной инжектированной последовательности преобразований новое событие инжектированного триггера принимается только в том случае, если оба бита JEOS мастер-АЦП и ведомой АЦП были очищены (фаза задержки). Любые новые события инжектированного триггера, происходящие во время текущей инжектированной последовательности и связанной фазы задержки, игнорируются. 
- После завершения регулярной последовательности преобразований мастер-АЦП новое событие регулярного триггера мастер-АЦП принимается только в том случае, если регистр данных мастера (ADC_DR) был прочитан. Любые новые события регулярного триггера, происходящие для мастер-АЦП во время текущей регулярной последовательности и связанных фаз задержки, игнорируются. 

Поведение аналогично для регулярных последовательностей, происходящих на ведомой АЦП.

## Режим одновременных регулярных преобразований (`DUAL[4:0]=0b00110`)
В этом режиме преобразования регулярной группы АЦП-мастера и АЦП-слейва выполняется от [внешнего триггера](#Преобразование%20по%20внешнему%20триггеру(EXTSEL,%20EXTEN,%20JEXTSEL,%20JEXTEN)) что мы выбрали в EXTSEL в АЦП-мастере. Инжектированные преобразования могут выполняться независимо. При этом запрос на инжектированные преобразования(не важно в АЦП-мастере или в АЦП-слейве) останавливают одновременные регулярные преобразования на обоих АЦП, а после инжектированных преобразований продолжаются дальше. 
> [!warning] 
> В регулярном одновременном режиме необходимо преобразовывать последовательности одинаковой длины или обеспечить, чтобы интервал между триггерами был длиннее, чем длиннейшее время преобразования из 2 последовательностей. В противном случае АЦП с самой короткой последовательностью может перезапуститься, пока АЦП с самой длинной последовательностью завершает предыдущие преобразования.

Программное обеспечение уведомляется прерываниями, когда оно может читать данные: 
- По завершении каждого события преобразования (EOC) на мастер-АЦП генерируется прерывание мастер-EOC (если включено EOCIE), и программное обеспечение может читать ADC_DR мастер-АЦП. 
- По завершении каждого события преобразования (EOC) на ведомой АЦП генерируется прерывание ведомой EOC (если включено EOCIE), и программное обеспечение может читать ADC_DR ведомой АЦП. 
- Если продолжительность регулярной последовательности мастера равна продолжительности регулярной последовательности ведомой, программное обеспечение может включить только одно из двух прерываний EOC (например, мастер EOC) и читать оба преобразованных данных из общего регистра данных (ADCx_CDR).

### Использование DMA в dual mode регулярных преобразований
В режиме одновременных регулярных преобразований возможно чтение регулярных данных с использовании DMA(конечно мы можем и в других режимах пользоваться DMA но там не будут одновременно работать конкретно регулярные преобразования). Возможны два метода:
1. Использование двух каналов DMA для двух АЦП(MDMA=0)
- Настройте канал DMA мастер-АЦП для чтения ADC_DR из мастера. Запросы DMA генерируются при каждом событии EOC мастер-АЦП.
- Настройте канал DMA ведомой АЦП для чтения ADC_DR из ведомой. Запросы DMA генерируются при каждом событии EOC ведомой АЦП.
2. Использования режима MDMA, который оставляет один канал DMA свободным для других использований(MDMA=0b10 для 12 или 10 битных АЦП или MDMA=0b11 для 8 или 6 битных АЦП)
- Используется один канал DMA (мастера). Настройте канал DMA мастер-АЦП для чтения общего регистра АЦП (ADCx_CDR). 
- Один запрос DMA генерируется каждый раз, когда происходят оба события EOC мастера и ведомой. В это время преобразованные данные ведомой АЦП доступны в верхнем полуслове 32-битного регистра ADCx_CDR, а преобразованные данные мастер-АЦП доступны в нижнем полуслове регистра ADCx_CDR. 
- Оба флага EOC очищаются, когда DMA читает регистр ADCx_CDR.
> [!warning] 
> В режиме MDMA обязательно нужно чтобы у обоих АЦП были одинаковый размер последовательности преобразований, потому что иначе не будет использоваться ADC_CDR

Если [DISCEN = 1](#Регулярных%20групп(DISCEN=1)), то каждые "n" одновременных преобразований регулярной последовательности требуют события регулярного триггера ("n" определяется DISCNUM). 
Этот режим может быть комбинирован с режимом AUTDLY: 
- После завершения одновременного преобразования последовательности следующее преобразование в последовательности начинается только в том случае, если ADCx_CDR (или ADC_DR мастер-АЦП), был прочитан (фаза задержки). 
- После завершения одновременной регулярной последовательности преобразований новое событие регулярного триггера принимается только в том случае, если общий регистр данных (ADCx_CDR) был прочитан (фаза задержки). Любые новые события регулярного триггера, происходящие во время текущей регулярной последовательности и связанных фаз задержки, игнорируются.

Возможно использование DMA для обработки данных в регулярном одновременном режиме, комбинированном с режимом AUTDLY, при условии, что используется режим мульти-DMA: биты MDMA должны быть установлены в 10 или 11. 
Когда регулярный одновременный режим комбинируется с режимом AUTDLY, пользователь должен обеспечить, чтобы: 
- Количество преобразований в последовательности мастера было равно количеству преобразований в последовательности ведомой. 
- Для каждого одновременного преобразования последовательности длина преобразования ведомой АЦП была меньше длины преобразования мастер-АЦП. Обратите внимание, что длина последовательности зависит от количества каналов для преобразования, времени выборки и разрешения каждого канала. 
> [!warning] 
>  Эта комбинация регулярного одновременного режима и режима AUTDLY ограничена случаем, когда программируются только регулярные каналы: запрещается программировать инжектированные каналы в этом комбинированном режиме.

## Регулярные каналы
1. [Выбираем источник тактирования для АЦП и делитель частоты тактирования](#Тактовый%20сигнал%20АЦП)
2. Включаем тактирование АЦП на шине(да, даже если источник тактового сигнала является SYSCLK, нам нужно тактирование на шине, чтобы мы могли менять регистры)
3. [Выходим из режима глубокого энергосбережения, сбросив бит DEEPPWD в ADC_CR](#Включение/выключение%20АЦП)
4. Включаем регулятор напряжения АЦП, установив бит ADVREGEN в ADC_CR и ждём время пока он включится
5. [Выполняем калибровку АЦП(ADCAL=1 в ADC_CR)  и ждём пока у нас автоматически бит ADCAL не сбросится(ну т.е. пока у нас не закончиться калибровка](#Калибровка%20АЦП)
6. [Выбираем какие каналы в АЦП будут дифференциальными или single-ended в регистре ADC_DIFSEL(по умолчанию все каналы single-ended)](#Одноконцевой(single-ended)%20и%20дифференциальный%20вход%20в%20каналах) 
7. Очищаем ADRDY в ADC_ISR регистре, записывая туда 1 
8. Включаем АЦП ADEN=1 в ADC_CR и ждём пока у нас автоматически не очистится флаг ADRDY в ADC_ISR(ну то есть пока у нас не включится АЦП)
9. Создаём последовательность преобразований каналов через ADC_SQRy
10. [Устанавливаем время выборки(ну т.е. зарядки конденсатора) в ADC_SMPRy](#Преобразования%20в%20группе)
11. [Выбираем режим преобразования в группе(одиночный/непрерывный CONT=0 или прерывистый DISCEN=1 в ADC_CFGR)](#Режимы%20преобразования%20в%20группе)
12. [Выбираем будем ли мы работать по программному или аппаратному триггеру через EXTSEL и если по аппаратному настраиваем что будет триггером в JEXTSEL](#Преобразование%20по%20внешнему%20триггеру(EXTSEL,%20EXTEN,%20JEXTSEL,%20JEXTEN))

<span style="color:grey">(опционально)</span>

12. [Включаем выравнивание битов преобразованных значений в ADC_DR по нужному краю через выставление ALIGN в ADC_CFGR](#Размер%20данных%20и%20выравнивание)
13. [Включаем смещение к нужному каналу y через OFFSET_EN в регистре ADC_OFRy](#Смещение)
14. [Включаем компенсацию усиления установкой GCOMP в ADC_CFGR2](#Компенсация%20усиления)
15. [Настраиваем, сохраняются ли данные или перезаписываются при возникновении события переполнения, программируя управляющий бит OVRMOD в ADC_CFGR](#Переполнение%20АЦП(OVR,%20OVRMOD))
16. [Включаем DMA(DMAEN=1 в ADC_CFGR)](#С%20DMA)
17. Выбираем режим работы DMA через DMACFG в ADC_CFGR
18. Включаем нужные прерывания в ADC_IER(это прерывания самого АЦП)
19. [Настраиваем сам DMA](DMA#Пример%20настройки%20DMA)
20. Настраиваем прерывания для DMA

<span style="color:grey">(опционально)</span>

21. Запускаем АЦП(ADSTART в ADC_CR)
22. Если ставили аппаратный триггер, то считывание начнётся когда вызовется аппаратный триггер. Если триггера нет, то преобразование начнётся сразу после ADSTART
23. Когда произойдёт преобразование одного канала из последовательности то поднимется флаг EOC(если поставлен EOCIE то вызовется прерывание)
24. Когда пройдёт преобразование всех каналов в последовательности то поднимется флаг EOS(если поставлен EOSIE то вызовется прерывание)

# Пример настройки
В главе GPIO можно узнать, что после сброса нигде не используемые ножки настроены на аналоговый режим. Основная причина этому - в аналоговом режиме GPIO потребляют меньше всего энергии и соответственно это упрощает настройку микроконтроллера в режим низкого энергопотребления.

Если воспользоваться пониманием этого аспекта, то настройка нужных ножек в аналоговый режим может быть осуществлена включением тактирования порта, без дополнительных настроек.
```c
RCC->AHB2ENR |= RCC_AHB2ENR_GPIOCEN;
```
Но дело в том что нам лучше делать полную настройку gpio, чтобы не путать читающего код:
```c
RCC->AHB2ENR |= RCC_AHB2ENR_GPIOCEN;

GPIOC->MODER &= ~(GPIO_MODER_Msk << GPIO_MODER_MODE0_Pos
				  | GPIO_MODER_Msk << GPIO_MODER_MODE1_Pos
				  | GPIO_MODER_Msk << GPIO_MODER_MODE2_Pos
				  | GPIO_MODER_Msk << GPIO_MODER_MODE3_Pos);

GPIOC->MODER |= (GPIO_ANALOG_IN_Mode << GPIO_MODER_MODE0_Pos
				 | GPIO_ANALOG_IN_Mode << GPIO_MODER_MODE1_Pos
				 | GPIO_ANALOG_IN_Mode << GPIO_MODER_MODE2_Pos
				 | GPIO_ANALOG_IN_Mode << GPIO_MODER_MODE3_Pos);
```

В примере ниже показана самая минимальная процедура настройка одного канала  АЦП и считывание данных из этого канала.
```c
uint16_t current_adc_value;

int main( void )
{
	RCC->AHB2ENR |= RCC_AHB2ENR_GPIOCEN;
	
	// Выбрать SYSCLK в качестве источника тактового сигнала для АЦП
	RCC->CCIPR |= (2U) << RCC_CCIPR_ADC12SEL_Pos;
	
	// Включить тактирование АЦП.
	RCC->AHB2ENR |= RCC_AHB2ENR_ADC12EN;
	
	// Выйти из режима power-down.
	ADC2->CR &= ~(ADC_CR_DEEPPWD);
	
	// Включить внутренний преобразователь питания и подождать завершения.
	ADC2->CR |= ADC_CR_ADVREGEN;
	dummy_delay( 10000 );
	
	// Включить автоматическую калибровку АЦП и подождать пока она закончится.
	ADC2->CR |= ADC_CR_ADCAL;
	while ( ADC2->CR & ADC_CR_ADCAL ){}
	
	// Включить АЦП.
	ADC2->ISR |= ADC_ISR_ADRDY; // Очистить флаг ADRDY.
	ADC2->CR |= ADC_CR_ADEN;
	while ( !(ADC2->ISR & ADC_ISR_ADRDY) ){} // Подождать до включения.
	
	// Увеличить время считывания показания для нужного нам канала
	ADC2->SMPR1 |= 2 << ADC_SMPR1_SMP6_Pos;
	
	// Настроить канал, из которого мы будем считывать данные.
	// Канал №6, всего каналов - 1
	ADC2->SQR1 |= 6 << ADC_SQR1_SQ1_Pos | 0 << ADC_SQR1_L_Pos;
	
	// Начать считывание данных из АЦП
	ADC2->CR |= ADC_CR_ADSTART;
	
	// Подождать до завершения измерения
	while ( !(ADC2->ISR & ADC_ISR_EOS) ){}
	
	// Сбросить флаг завершения измерения
	ADC2->ISR |= ADC_ISR_EOS;
	
	// Прочитать данные из АЦП
	current_adc_value = ADC2->DR;
	
	while ( 1 )
	{}
}
```

Так вот тут интересно.
`ADC2->CR` используется для управления самим АЦП2

`ADVREGEN` включает внутренний регулятор питания, что даёт более стабильное и точное напряжение питания для работы АЦП
