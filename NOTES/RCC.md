# Настройка тактовой частоты
Все шины получают тактовый сигнал от системного источника тактового сигнала (SYSCLK), который в свою очередь может получать сигнал от:
- Внутреннего RC-генератора частотой 16 MHz (HSI) - справедливо для большинства семейств STM32;
- Внешнего кварцевого резонатора частоты от 4 до 48 MHz (HSE);
- Генератора с фазовой автоподстройкой частоты (PLL) - устройства, позволяющего умножать или делить частоту на заданный целый коэффициент, чтобы получить желаемое значение.
Чем больше потребление тока, тем больше частоты, на которой работает микроконтроллер. В описании микроконтроллеров обычно используется характеристика мА/МГц.

При подаче питания микроконтроллеры STM32G4 работают от внутреннего генератора, который хорош тем, что не требует для использования никаких других компонентов, тем самым не требуя лишних компонентов, однако данный генератор не является точным и его частота может значительно изменяться от температуры.

Микроконтроллеры STM32G4 могут работать на частоте до 170 MГц. Для ее получения источником частоты SYSCLK должен быть генератор с фазовой автоподстройкой частоты - ФАПЧ (PLL). Источником же тактовой частоты для PLL может служить либо HSI, либо HSE. PLL осуществляет умножение частоты в несколько этапов, что обеспечивает гибкую настройку вне зависимости от частоты имеющегося источника.
## Примеры
### Настройка тактовой частоты системы напрямую от внешнего резонатора
```c
int main(void)
{
	// Включить внешний резонатор
	RCC->CR |= RCC_CR_HSEON;
	
	// Подождать, пока резонатор запустится
	while ((RCC->CR & RCC_CR_HSERDY)!= RCC_CR_HSERDY) {}
	
	// Использовать HSE в качестве источника SYSCLK 
	RCC->CFGR |= RCC_CFGR_SW_Msk;
	RCC->CFGR &= ~ (RCC_CFGR_SW_Msk ^ RCC_CFGR_SW_HSE);
	
	// Подождать до момента, пока источник частоты поменяется
	while ((RCC->CFGR & RCC_CFGR_SWS_HSE) != RCC_CFGR_SWS_HSE) {}
	
	// ... Основная программа
}
```

```c
// Включить внешний резонатор
RCC->CR |= RCC_CR_HSEON;

// Подождать, пока резонатор запустится
while ((RCC->CR & RCC_CR_HSERDY)!= RCC_CR_HSERDY) {}
```

*глава 7.4.1 стр. 290*
Регистр `RCC->CR` (RCC Control Register) в микроконтроллерах используется для управления включением и настройкой внешних источников тактирования.

в RCC->CR есть два бита для включение таймера HSE(`HSEON`) и для того чтобы узнать включился ли он(`HSERDY`). Дело в том что в когда устанавливается бит `HSEON` в регистре `RCC->CR`, микроконтроллер начинает включать внешний источник тактового сигнала HSE. Однако, этот процесс может занять некоторое время, и бит `HSERDY` (High-Speed External oscillator Ready) в регистре `RCC->CR` станет равным 1 только после того, как внешний источник будет полностью готов.
Поэтому, цикл `while ((RCC->CR & RCC_CR_HSERDY) != RCC_CR_HSERDY) {}` ожидает, пока бит `HSERDY` не станет равным 1.

```c
// Использовать HSE в качестве источника SYSCLK 
RCC->CFGR |= RCC_CFGR_SW_Msk;
RCC->CFGR &= ~ (RCC_CFGR_SW_Msk ^ RCC_CFGR_SW_HSE);
```
![[Pasted image 20231215213337.png]]
Регистр `RCC_CFGR` (RCC Configuration Register) в микроконтроллерах STM32 отвечает за настройку различных параметров системы тактирования. Этот регистр позволяет выбирать источники тактового сигнала для различных системных блоков микроконтроллера.

Некоторые из битовых полей в регистре `RCC_CFGR` включают:
- **SW (System clock Switch):** Это поле определяет источник системного тактового сигнала (`SYSCLK`). Значение этого поля выбирается так, чтобы определить, откуда будет браться основной тактовый сигнал (например, HSI, HSE, или PLL).
- **HPRE (AHB prescaler):** Это поле определяет делитель для AHB (Advanced High-performance Bus), который контролирует, как быстро тактовый сигнал будет подаваться на шину AHB. Это влияет на тактирование многих периферийных устройств.
- **PPRE1, PPRE2 (APB prescaler):** Эти поля определяют делители для шин APB1 и APB2 соответственно. Они контролируют, как быстро тактовый сигнал будет подаваться на шины APB1 и APB2, и, таким образом, влияют на тактирование различных периферийных устройств, подключенных к этим шинам.

Настройка регистра CFGR на использование HSE в качестве источника SYSCLK в данном примере отличается от привычного подхода, который мы использовали для установки регистров. Вместо того, чтобы сначала очистить регистр, а потом заполнить его нужными данными, мы сначала заполняем нужное поле единицами (используя маску поля), а потом оставляем только нужные нам биты используя XOR маски поля и нужной нам маски. Это необходимо делать из-за того, что значение данной группы бит (SW) не может быть равно 0 и автоматически устанавливается в 0b01, соответственно при классическом подходе появляются проблемы с установкой значений.

```c
// Подождать до момента, пока источник частоты поменяется
while ((RCC->CFGR & RCC_CFGR_SWS_HSE) != RCC_CFGR_SWS_HSE) {}
```

Биты `SWS` (System Clock Switch Status) в регистре `RCC_CFGR` используются для указания текущего выбранного источника тактового сигнала системы (SYSCLK). Значение `RCC_CFGR_SWS_HSE` указывает, что в настоящее время системный тактовый сигнал (SYSCLK) происходит от внешнего источника HSE.
### Настройка тактовой частоты на 64 МГц с использованием PLL с HSE в качестве источника для PLL.
У HSE изначально частота 8 МГц, по сему мы используем умножение на 32 и деление на 4, (PLLM=1, PLLR = 0)$8"МГц"*3 2/4=6 4"Мгц"$ 
<span style="color: red; font-size: 20">Максимальная частота 170 МГц!</span>

```c
void setup_clock(void){
    // Установить задержку на доступ к Flash
    FLASH->ACR &= ~FLASH_ACR_LATENCY_Msk;
    FLASH->ACR |= FLASH_ACR_LATENCY_2WS;
    
    // Включить HSE как и в прошлом примере
    RCC->CR |= RCC_CR_HSEON;
    while ((RCC->CR & RCC_CR_HSERDY) != RCC_CR_HSERDY) {}
    
    // Сбросить начальные значения регистра настройки PLL
    RCC->PLLCFGR &= ~(RCC_PLLCFGR_PLLR_Msk | RCC_PLLCFGR_PLLM_Msk | RCC_PLLCFGR_PLLN_Msk);
    
    // Настроить PLL по значениям, полученными в ioc файле
    RCC->PLLCFGR |= 0 << RCC_PLLCFGR_PLLR_Pos // Установка делителя R
                    | RCC_PLLCFGR_PLLREN    // Включение R делителя PLL
                    | 32 << RCC_PLLCFGR_PLLN_Pos // Установка умножителя N
                    | 1 << RCC_PLLCFGR_PLLM_Pos // Установка делителя M
                    | RCC_PLLCFGR_PLLSRC_HSE; // HSE - источник сигнала для PLL
    
    // Включить PLL
    RCC->CR |= RCC_CR_PLLON;
    
    // Подождать, пока PLL включится
    while ((RCC->CR & RCC_CR_PLLRDY) != RCC_CR_PLLRDY){}
    
    // Использовать PLL в качестве источника SYSCLK
    RCC->CFGR |= RCC_CFGR_SW_Msk;
    RCC->CFGR &= ~(RCC_CFGR_SW_Msk ^ RCC_CFGR_SW_PLL);
    
    // Подождать, до момента, пока источник частоты поменяется
    while ((RCC->CFGR & RCC_CFGR_SWS_PLL) != RCC_CFGR_SWS_PLL){}
}
```

```c
// Установить задержку на доступ к Flash
FLASH->ACR &= ~FLASH_ACR_LATENCY_Msk;
FLASH->ACR |= FLASH_ACR_LATENCY_2WS;
```

*гл. 3.7.19 стр. 146*
- `FLASH` - это структура, предоставляющая доступ к управлению Flash-памятью.
- `ACR` - это регистр управления доступом к Flash-памяти (`Access Control Register`).
- `FLASH_ACR_LATENCY_Msk` - маска для битов, отвечающих за количество тактов задержки доступа к Flash. В данном случае, `LATENCY` - это биты отвечающие за задержку.
- `FLASH_ACR_LATENCY_2WS` - это значение, устанавливающее количество тактов задержки доступа к Flash в два такта.
В этом примере мы кроме самого тактирования через PLL настраиваем еще и доступ к flash памяти (памяти, в которой хранится код программы). Это необходимо из-за того, что flash память имеет низкую частоту считывания данных. Соответственно, если частота процесса слишком высокая, данные из flash не будут успевать считываться за один такт и микроконтроллер не сможет выполнять программу. Чтобы исправить эту проблему мы настраиваем искусственную задержку на считывание данных из FLASH. То есть каждый раз, когда микроконтроллер будет обращаться во FLASH память, считывание будет занимать указанное количество тактов. Таблица с величиной необходимой задержки в зависимости от частоты показана на рисунке 4.3.4. В лабораторных работах мы всегда работаем со стандартным режимом - Vcore Range 1 normal mode.

// Сбросить начальные значения регистра настройки PLL
RCC->PLLCFGR &= ~(RCC_PLLCFGR_PLLR_Msk | RCC_PLLCFGR_PLLM_Msk | RCC_PLLCFGR_PLLN_Msk);

`RCC->PLLCFGR` отвечает за настройку параметров фазовой блокировки (PLL - Phase-Locked Loop).
![[Pasted image 20231215215446.png]]
*гл. 7.4.4 стр. 294*
1) `RCC_PLLCFGR_PLLR_Msk`: Эта маска используется для выбора битов, отвечающих за делитель R в настройках PLL. PLLR - это делитель, который уменьшает выходную частоту PLL.
2) `RCC_PLLCFGR_PLLM_Msk`: Эта маска используется для выбора битов, отвечающих за делитель M в настройках PLL. PLLM - это делитель, который уменьшает входную частоту PLL.
3) `RCC_PLLCFGR_PLLN_Msk`: Эта маска используется для выбора битов, отвечающих за множитель N в настройках PLL. PLLN - это множитель, который увеличивает входную частоту PLL.

```c
// Настроить PLL по значениям, полученными в ioc файле
RCC->PLLCFGR |= 0 << RCC_PLLCFGR_PLLR_Pos // Установка делителя R
				| RCC_PLLCFGR_PLLREN    // Включение R делителя PLL
				| 32 << RCC_PLLCFGR_PLLN_Pos // Установка умножителя N
				| 1 << RCC_PLLCFGR_PLLM_Pos // Установка делителя M
				| RCC_PLLCFGR_PLLSRC_HSE; // HSE - источник сигнала для PLL

// Включить PLL
RCC->CR |= RCC_CR_PLLON;
```
1) `RCC_PLLCFGR_PLLR_Pos`: Этот макрос представляет собой позицию бита в регистре PLLCFGR, который отвечает за делитель R. В данном случае, `0 << RCC_PLLCFGR_PLLR_Pos` устанавливает делитель R в 0, что означает деления на 2.
![[Pasted image 20240131202106.png]]
*стр 295*
3) `RCC_PLLCFGR_PLLREN`: Этот бит включает делитель R PLL. Если этот бит установлен в 1, то делитель R будет использоваться.
- `32 << RCC_PLLCFGR_PLLN_Pos`: Этот участок кода устанавливает множитель N PLL в 32. Множитель N увеличивает входную частоту PLL. Значение 32 означает, что частота увеличится в 32 раза.
![[Pasted image 20240131202138.png]]

1) `1 << RCC_PLLCFGR_PLLM_Pos`: Этот участок кода устанавливает делитель M PLL в 1. Делитель M уменьшает входную частоту PLL.

# Включение тактирования
Тактирование (clocking) в контексте микроконтроллеров и микропроцессоров означает предоставление основного временного сигнала (тактового сигнала), который синхронизирует работу всех внутренних компонентов и операций в микроконтроллере. Этот тактовый сигнал определяет скорость выполнения инструкций и операций внутри микроконтроллера.
Изначально везде тактовый сигнал стоит на аналоговом(то есть никаком), поэтому для работы с регистрами нужно изначально поставить тактовый сигнал.
Для этого нужно зайти на 83 страницу и найти где находится GPIOx, где x - буква нужного регистра с устройствами. И понять на какой шине они находятся.
Затем находим в 7.4 нужный нам регистр RCC для той или иной шины, и находим какой бит отвечает за нужный нам GPIOx.
## Пример
Мы хотим включить тактирование для портов A и D(GPIOA, GPIOD). Смотрим в 83 странице:

int main(void)
{

    // OR

    while(1)

    }
}
